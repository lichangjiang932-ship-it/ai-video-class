## 如何贡献

### 1. 报告内容错误

发现课程内容有误？请通过 [GitHub Issues](https://github.com/lichangjiang932-ship-it/ai-video-class/issues) 提交：

```
标题：[错误] 模块XX - 简要描述
内容：
- 文件名：模块XX-xxx.md
- 错误位置：第XX行 / 第X.X节
- 错误内容：（原文）
- 正确内容：（建议修改）
- 依据：（参考资料/实际测试结果）
```

### 2. 提出改进建议

有好的想法？同样通过 Issue 提交：

```
标题：[建议] 简要描述
内容：
- 建议类型：内容补充 / 结构优化 / 新增模块 / 其他
- 详细说明：（具体建议）
- 预期效果：（改进后的好处）
```

### 3. 提交内容修改

如果你熟悉 Git 操作，可以直接提交 Pull Request：

```bash
# 1. Fork 本仓库
# 2. 创建你的分支
git checkout -b feature/你的改进

# 3. 修改内容
# 4. 提交更改
git commit -m "描述你的修改"

# 5. 推送到你的分支
git push origin feature/你的改进

# 6. 创建 Pull Request
```

### PR 规范

- **标题**：`[类型] 简要描述`
  - `[修复]` 修正错误
  - `[补充]` 新增内容
  - `[优化]` 改进现有内容
  - `[格式]` 格式调整
- **描述**：说明修改原因和内容
- **关联 Issue**：如果有相关 Issue，请在 PR 中引用

## 内容规范

### Markdown 格式

- 使用中文标点符号
- 标题层级清晰（H1 → H2 → H3）
- 表格对齐整齐
- 代码块使用正确的语言标记
- 链接使用相对路径

### 内容质量

- 确保信息准确、有据可查
- 新增内容需有实际价值
- 避免重复已有内容
- 保持客观中立的语气

### 工具信息

- 工具名称使用官方名称
- 附上官网链接
- 标注价格信息（免费/付费/免费额度）
- 标注最后验证日期
