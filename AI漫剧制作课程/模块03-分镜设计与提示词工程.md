## 第13课：15秒分镜设计基础——把脚本变成画面

### 学习目标

1. 理解15秒视频片段的分镜设计原则
2. 掌握将播客对话脚本拆分为镜头的方法
3. 学会设计每个镜头的"首帧"和"尾帧"
4. 能独立完成一集播客的分镜表

### 一、为什么是15秒？

**即梦AI的15秒限制**：

即梦AI目前生成的视频片段最长约15秒。这意味着：
- 每个镜头必须控制在15秒以内
- 需要把长对话拆分成多个短片段
- 每个片段要有独立的"开始画面"和"结束画面"
- 片段之间需要通过首尾帧衔接

**15秒能装多少内容？**

| 内容类型 | 15秒可容纳量 |
|---------|-------------|
| 对话 | 2-4轮（每人说1-2句） |
| 独白 | 50-80字 |
| 动作 | 1个完整的动作序列 |
| 表情变化 | 1-2个情绪转折 |
| 场景转换 | 1次（通过首尾帧过渡） |

### 二、播客场景的镜头类型

宠物播客漫剧的核心场景是"两个角色对着麦克风聊天"，但画面不能单调。以下是常用的镜头类型：

| 镜头类型 | 画面内容 | 时长建议 | 适用场景 |
|---------|---------|---------|---------|
| 双人全景 | 豆包和奶盖同框，展示播客环境 | 3-5秒 | 开场、转场 |
| 豆包中景 | 豆包半身，对着镜头/麦克风说话 | 5-10秒 | 豆包发言时 |
| 奶盖中景 | 奶盖半身，对着镜头/麦克风说话 | 5-10秒 | 奶盖发言时 |
| 豆包特写 | 豆包面部，捕捉表情变化 | 3-5秒 | 豆包犯傻/感动时 |
| 奶盖特写 | 奶盖面部，捕捉表情变化 | 3-5秒 | 奶盖吐槽/微笑时 |
| 反应镜头 | 一方听到另一方话后的反应 | 2-3秒 | 吐槽/惊讶时 |
| 空镜 | 播客环境细节（麦克风、耳机等） | 2-3秒 | 转场、缓冲 |
| 想象画面 | 豆包/奶盖描述的场景（画中画） | 5-10秒 | 讲故事时 |

### 三、脚本拆分为镜头的方法

#### 3.1 拆分原则

**原则1：每轮对话换镜头**
- 豆包说话 → 豆包镜头
- 奶盖说话 → 奶盖镜头
- 但不需要每句话都换，连续2-3句可以保持同一镜头

**原则2：情绪变化换镜头**
- 从笑切换到认真 → 换镜头
- 从吐槽切换到走心 → 换镜头
- 情绪的"转折点"是换镜头的最佳时机

**原则3：笑点用特写强化**
- 笑点出现时，切到说话人的特写
- 或切到另一个人的"反应镜头"
- 特写能放大笑点的效果

**原则4：每个镜头有独立的"故事"**
- 首帧：这个镜头开始时的画面状态
- 尾帧：这个镜头结束时的画面状态
- 从首帧到尾帧要有变化（表情变化、动作变化、视线变化）

#### 3.2 拆分实操示例

以下面这段脚本为例：

```
豆包：（热情满满）奶盖！你今天也超好看的！
奶盖：（冷淡）……你每天都说这句话。
豆包：因为每天都超好看啊！
奶盖：（叹气）行吧，今天聊什么？
豆包：诶，我问你啊——你有没有那种社死的经历？
奶盖：（警觉）你想说什么？
```

**拆分为4个镜头**：

| 镜头编号 | 时长 | 首帧 | 尾帧 | 画面内容 |
|---------|------|------|------|---------|
| S01 | 5s | 豆包微笑张嘴 | 豆包说完话期待地看奶盖 | 豆包中景，热情开场 |
| S02 | 4s | 奶盖面无表情 | 奶盖叹气 | 奶盖中景，冷淡回应 |
| S03 | 3s | 豆包笑容更灿烂 | 豆包凑近镜头 | 豆包特写，"因为每天都超好看" |
| S04 | 5s | 豆包坏笑 | 奶盖警觉表情 | 双人全景，引出话题 |

### 四、首尾帧设计的核心概念

**什么是首尾帧？**

在即梦AI的"图生视频"工作流中：
- **首帧**：视频开始时的画面（用文生图/图生图生成的静态图）
- **尾帧**：视频结束时的画面（用文生图/图生图生成的静态图）
- **中间过程**：即梦AI根据首尾帧自动生成过渡动画

**为什么要设计首尾帧？**

- 确保视频片段的起点和终点是你想要的画面
- 控制镜头的"运动方向"（从A状态到B状态）
- 实现多个片段之间的无缝衔接（上一个镜头的尾帧 = 下一个镜头的首帧）

**首尾帧设计的基本要求**：

| 要求 | 说明 |
|------|------|
| 主体一致 | 首帧和尾帧中的角色必须是同一个人 |
| 动作连贯 | 尾帧应该是首帧动作的"自然延续" |
| 视线一致 | 角色的视线方向在首尾帧中保持一致（除非需要转头） |
| 背景一致 | 同一场景的首尾帧背景应该相同 |
| 情绪递进 | 尾帧的情绪应该是首帧情绪的"发展" |

### 五、分镜表模板

以下是标准的分镜表格式：

```
═══════════════════════════════════════════════════════
《豆包和奶盖的日常》第X集：[集名]
分镜表
═══════════════════════════════════════════════════════

镜头编号：S01
时长：5秒
镜头类型：豆包中景
首帧描述：[详细描述视频开始时的画面]
尾帧描述：[详细描述视频结束时的画面]
对话/旁白：[这个镜头中的台词]
情绪：[核心情绪]
运镜：[镜头运动方式，如"固定""缓慢推进""轻微左移"]

───────────────────────────────────────────────────────

镜头编号：S02
...
```

### 六、常见问题

**Q1：15秒能说多少台词？**
> 正常语速约每秒3-4个字，15秒约45-60个字。但考虑到表情和停顿，实际能装2-4轮对话。

**Q2：首尾帧的差异应该多大？**
> 差异太小（几乎一样）→ 视频看起来像静止的
> 差异太大（完全不同）→ AI生成的过渡会很突兀
> 建议：同一场景内，首尾帧的差异控制在"一个表情变化"或"一个动作"的范围内。

**Q3：一个镜头一定要15秒吗？**
> 不一定。15秒是上限。3秒、5秒、8秒的镜头都可以。关键是节奏感——快节奏用短镜头，慢节奏用长镜头。

### 七、课后练习

1. **基础练习**：将一段5轮对话的脚本拆分为4-6个镜头
2. **进阶练习**：为每个镜头设计首帧和尾帧描述
3. **挑战练习**：设计一个包含"想象画面"（画中画）的分镜方案

## 第14课：即梦AI提示词工程——精准描述你的画面

### 一、提示词六要素

一个高质量的即梦AI提示词由六个核心要素组成：

```
[主体描述] + [动作表情] + [环境场景] + [光线氛围] + [镜头构图] + [风格质量]
```

**权重规则**：AI对提示词中靠前的内容赋予更高权重。最重要的信息放最前面。

## 第15课：首尾帧衔接设计——让15秒片段无缝连接

### 学习目标

1. 理解首尾帧衔接的核心原理
2. 掌握四种常用的首尾帧衔接模式
3. 学会设计多人对话场景的镜头切换方案
4. 能独立完成一集播客的首尾帧衔接表

### 一、首尾帧衔接的核心原理

**什么是首尾帧衔接？**

在即梦AI的图生视频工作流中：
- 你先生成"首帧"（静态图片）
- 再生成"尾帧"（静态图片）
- 即梦AI根据首尾帧自动生成中间的过渡视频

**衔接的关键**：
- 上一个镜头的**尾帧**，就是下一个镜头的**首帧**
- 这样镜头之间就不会有"跳跃感"
- 整集视频看起来是连贯的

**示意**：
```
镜头1：首帧A → [即梦AI生成过渡] → 尾帧B
镜头2：首帧B → [即梦AI生成过渡] → 尾帧C
镜头3：首帧C → [即梦AI生成过渡] → 尾帧D
...
```

### 二、四种常用衔接模式

#### 模式A：同场景同角色——表情/动作变化

**适用场景**：豆包从微笑变成大笑

```
首帧：豆包微笑，嘴巴微张正在说话
尾帧：豆包咧嘴大笑，眼睛弯成月牙
```

**提示词变化**：
- 首帧提示词：`...微笑的表情，嘴巴微张正在说话...`
- 尾帧提示词：`...开心地大笑，眼睛弯成月牙，露出洁白的牙齿...`

**即梦AI会自动生成**：豆包从微笑逐渐变成大笑的过渡动画。

#### 模式B：同场景不同角色——正反打切换

**适用场景**：豆包说完话，切到奶盖的反应

```
镜头1尾帧：豆包说完话，期待地看向画面右侧（看向奶盖的方向）
镜头2首帧：奶盖面无表情，视线朝向画面左侧（看向豆包的方向）
```

**关键设计**：
- 豆包的尾帧中，视线方向朝右
- 奶盖的首帧中，视线方向朝左
- 这样观众感觉两人在"对视"

#### 模式C：同场景全景→特写——景别变化

**适用场景**：从两人全景切到一人特写

```
镜头1尾帧：双人全景，豆包和奶盖在画面中
镜头2首帧：豆包特写，面部充满画面
```

**设计要点**：
- 特写首帧中，豆包的位置和表情要与全景尾帧中一致
- 只是"镜头推近"了，其他都没变

#### 模式D：场景转换——通过动作/视线过渡

**适用场景**：从播客现场切到"想象画面"

```
镜头1尾帧：豆包闭上眼睛，开始"回忆"
镜头2首帧：想象画面中的豆包（不同场景、不同光线）
```

**设计要点**：
- 尾帧用"闭眼""转头""看向窗外"等动作暗示场景切换
- 首帧的光线和氛围要有明显变化（提示观众这是不同场景）

### 三、播客场景的镜头切换方案

**标准播客对话的镜头切换模式**：

```
开场全景 → 豆包中景 → 奶盖中景 → 豆包特写（笑点）→ 奶盖反应 → 双人全景
    ↓           ↓           ↓           ↓              ↓           ↓
  建立环境    豆包发言    奶盖回应    笑点强化      吐槽反应    回到全景
```

**具体切换方案**：

| 对话阶段 | 镜头类型 | 切换时机 | 首尾帧要点 |
|---------|---------|---------|-----------|
| 开场 | 双人全景 | 固定5秒 | 展示完整环境 |
| 豆包发言 | 豆包中景 | 豆包开口时切 | 首帧：豆包微笑；尾帧：豆包说完看奶盖 |
| 奶盖回应 | 奶盖中景 | 奶盖开口时切 | 首帧：奶盖面无表情；尾帧：奶盖吐槽完 |
| 笑点爆发 | 豆包/奶盖特写 | 笑点出现时切 | 首帧：正常表情；尾帧：大笑/无语 |
| 反应镜头 | 另一方中景 | 吐槽后切 | 首帧：听对方说话；尾帧：自己的反应 |
| 转场 | 空镜/全景 | 话题切换时 | 展示环境细节或回到全景 |

### 五、完整示例：一集播客的首尾帧设计

以下是一集播客开场段落的完整首尾帧设计：

```
【开场段落脚本】
豆包：（热情满满）奶盖！你今天也超好看的！
奶盖：（冷淡）……你每天都说这句话。
豆包：因为每天都超好看啊！
奶盖：（叹气）行吧，今天聊什么？
```

**S01：双人全景（3秒）**
- 首帧：播客录音间全景，豆包和奶盖坐在桌前，两人还没开始说话，微笑着面对镜头
- 尾帧：豆包转向奶盖方向，嘴巴微张准备说话
- 衔接方式：动作过渡（静止→开始说话）

**S02：豆包中景（4秒）**
- 首帧：豆包微笑，嘴巴微张正在说话（= S01尾帧中豆包的状态）
- 尾帧：豆包说完话，期待地看着画面右侧（奶盖方向）
- 衔接方式：模式A（表情从说话变成期待）

**S03：奶盖中景（4秒）**
- 首帧：奶盖面无表情，视线朝画面左侧（豆包方向）
- 尾帧：奶盖叹气，微微摇头
- 衔接方式：模式B（正反打，从豆包切到奶盖）

**S04：豆包特写（3秒）**
- 首帧：豆包灿烂笑容，眼睛弯成月牙（回应"因为每天都超好看"）
- 尾帧：豆包笑容更灿烂，带着一点得意
- 衔接方式：模式C（从奶盖中景切到豆包特写）

**S05：奶盖特写（3秒）**
- 首帧：奶盖无语的表情，眼神放空（听到豆包的话后的反应）
- 尾帧：奶盖嘴角微微上扬（忍不住被逗笑）
- 衔接方式：模式B（正反打，从豆包特写切到奶盖特写）

### 六、常见问题

**Q1：首尾帧的差异太小，视频看起来像静止的怎么办？**
> 增大首尾帧之间的差异。比如首帧是微笑，尾帧改成大笑；或者首帧是正面，尾帧改成微微侧头。

**Q2：AI生成的过渡动画不自然怎么办？**
> 确保首尾帧的主体位置、光线、背景一致。差异只在表情或动作上。如果背景差异太大，AI会生成不自然的过渡。

**Q3：正反打切换时，两人的视线方向对不上怎么办？**
> 严格遵守"左看右"的规则：画面左侧的角色视线朝右，画面右侧的角色视线朝左。

2. **进阶练习**：设计一个包含景别变化（全景→特写→全景）的衔接方案
3. **挑战练习**：设计一个包含"场景转换"（播客现场→想象画面→回到现场）的衔接方案

### 六、模板使用方法

**Step 1：选择模板**
根据镜头需要，从模板库中选择最接近的模板。

**Step 2：修改变量**
- 修改表情描述（把"微笑"改成"大笑"等）
- 修改动作描述（把"坐着"改成"前倾"等）
- 保留固定词（角色外貌、环境、风格）

**Step 3：调整权重**
- 把最重要的信息放在提示词最前面
- 次要信息放后面

**Step 4：生成并迭代**
- 用即梦AI生成4张，选择最佳
- 不满意就微调提示词重新生成

### 七、常见问题

**Q3：模板需要经常更新吗？**

### 四、质量检查清单

**分镜设计检查**：
- [ ] 每个镜头时长是否在15秒以内？
- [ ] 镜头类型是否有变化（不全是中景）？
- [ ] 笑点处是否有特写强化？
- [ ] 走心段落是否有足够的镜头铺垫？

**首尾帧检查**：

- [ ] 正反打切换时视线方向是否正确？
- [ ] 同一角色在不同镜头中外貌是否一致？
- [ ] 背景和光线是否一致？

**提示词检查**：
- [ ] 角色固定词是否每次都出现？
- [ ] 提示词长度是否在150字以内？
- [ ] 最重要的信息是否在前面？
- [ ] 风格描述是否统一？

### 五、常见问题

**Q1：41个镜头太多了，可以减少吗？**
> 可以。合并一些短镜头，比如连续3轮对话可以用2个镜头（豆包3句话+奶盖3句话）而不是6个镜头。

**Q2：首尾帧都要写提示词，工作量翻倍了怎么办？**
> 同一镜头的首尾帧提示词80%是相同的，只有表情/动作部分不同。可以先写一个"基础版"，然后复制修改表情部分。

**Q3：分镜设计好了但实际生成时发现不行怎么办？**
> 这很正常。分镜是"计划"，实际生成是"执行"。预留20%的时间用于调整。

## 模块三总结

### 核心知识点回顾

1. **15秒分镜原则**：每个镜头控制在15秒以内，2-4轮对话为一个镜头
2. **镜头类型**：双人全景、单人中景、情绪特写、反应镜头、空镜转场
3. **脚本拆分**：按对话轮次、情绪变化、笑点位置拆分镜头
4. **首尾帧衔接**：上一镜头尾帧=下一镜头首帧，确保无缝过渡
5. **提示词六要素**：主体+动作+环境+光线+构图+风格
6. **角色一致性**：固定词每次必须出现，可变词根据场景调整
7. **模板库**：按角色+情绪分类，快速调用和修改
8. **正反打切换**：左看右、右看左，视线方向要对上
