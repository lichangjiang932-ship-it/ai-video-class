## 第35课：角色一致性核心——让豆包和奶盖永远"不换脸"

### 学习目标

1. 理解AI图像生成中角色不一致的根本原因
2. 掌握即梦AI中保持角色一致性的三大核心方法
3. 能独立为豆包和奶盖建立角色一致性参考体系
4. 能在不同场景中稳定生成同一角色形象

---

### 一、为什么AI总会"换脸"

#### 1.1 角色不一致的根本原因

每次在即梦AI中输入提示词生成图片，模型都是"从零开始"理解你的描述。即使你写了完全相同的提示词，以下因素也会导致角色外貌偏移：

| 原因 | 表现 | 发生概率 |
|------|------|---------|
| 随机种子差异 | 每次生成的底层随机数不同 | 100% |
| 提示词歧义 | "金毛犬"可以是无数种金毛犬 | 高 |
| 缺乏视觉锚点 | 没有参考图，AI自由发挥 | 高 |
| 模型理解偏差 | AI对"豆包"没有记忆 | 100% |

**核心结论**：AI没有"记忆"，每次生成都是独立事件。要保持一致性，必须用**视觉锚点**锁定角色特征。

#### 1.2 一致性对宠物拟人漫剧的特殊挑战

宠物拟人角色比人类角色更难保持一致：

- **品种特征模糊**：金毛犬之间差异微妙，不像人类有明显五官区别
- **拟人化程度不稳定**：有时更像人，有时更像动物
- **服装和配饰是关键区分点**：宠物角色需要靠配饰来"认人"
- **表情和肢体语言差异大**：不同生成中嘴型、耳朵角度可能完全不同

### 二、方法一：角色种子锁定法

#### 2.1 什么是种子锁定

每张AI生成的图片背后都有一个"随机种子"（Seed）数字。相同的提示词+相同的种子=相同（或高度相似）的结果。

#### 2.2 操作步骤

**Step 1：生成满意的豆包基准图**

在即梦AI中输入提示词：
```
一只可爱的金毛犬，拟人化站立姿态，穿着蓝色卫衣和卡其色工装裤，
戴着一副黑框眼镜，表情温暖微笑，大眼睛，毛发蓬松金色，
竖屏9:16比例，电影质感，柔和的演播室灯光
```

生成4张图片，选择最满意的作为豆包基准图。

**Step 2：记录种子值**

- 在即梦AI的生成结果详情中找到"Seed"值
- 记录下来，例如：`豆包种子 = 1234567890`

**Step 3：后续生成固定使用该种子**

在生成豆包的新场景图片时，在高级设置中手动填入该种子值：
```
提示词：豆包坐在播客麦克风前，表情认真倾听...
高级设置 → Seed: 1234567890
```

#### 2.3 种子法的优缺点

| 优点 | 缺点 |
|------|------|
| 操作简单，不需要额外工具 | 不同场景下可能仍有偏差 |
| 不消耗额外积分 | 角度和姿态变化有限 |
| 对同一场景复现效果好 | 无法保证跨风格一致性 |

#### 2.4 豆包和奶盖的种子记录表

```
角色    | 基准图种子     | 适用风格    | 备注
--------|---------------|------------|------
豆包    | 1234567890    | 写实动漫    | 蓝色卫衣+黑框眼镜
奶盖    | 9876543210    | 写实动漫    | 粉色针织衫+蝴蝶结
豆包    | 1122334455    | 水彩插画    | 同上配饰
奶盖    | 5566778899    | 水彩插画    | 同上配饰
```

### 三、方法二：参考图喂养法（核心方法）

#### 3.1 原理

将之前生成的满意角色图片作为"参考图"上传给即梦AI，让AI在新生成中参照这张图片的角色特征。

#### 3.2 操作步骤

**Step 1：准备豆包的角色参考图集**

至少准备以下角度的参考图：

| 角度 | 用途 | 提示词要点 |
|------|------|-----------|
| 正面全身 | 主参考图 | 播客场景标准站姿 |
| 3/4侧面 | 对话镜头 | 面朝右侧，露出半边脸 |
| 正面半身 | 特写镜头 | 胸部以上，表情清晰 |
| 背面 | 过渡镜头 | 背对镜头，配饰可见 |

**Step 2：在即梦AI中使用"以图生图"功能**

1. 选择"以图生图"或"参考图"模式
2. 上传豆包的正面全身参考图
3. 输入新场景的提示词：
```
这只金毛犬坐在播客桌前，面前摆着麦克风和耳机，
表情兴奋地说话，背景是温馨的播客工作室，
竖屏9:16，电影质感灯光
```
4. 调整"参考强度"为70-80%（太高会限制姿态变化，太低会丢失角色特征）

**Step 3：检查生成结果**

对照以下清单检查：
- [ ] 毛发颜色是否为金色？
- [ ] 是否戴着眼镜？
- [ ] 穿着是否为蓝色卫衣？
- [ ] 体型是否与参考图一致？
- [ ] 面部特征是否相似？

#### 3.3 参考强度的黄金比例

| 参考强度 | 效果 | 适用场景 |
|---------|------|---------|
| 50-60% | 轻度参考，姿态自由度大 | 大幅度动作变化 |
| 70-80% | **推荐**，平衡一致性与自由度 | 大多数场景 |
| 85-95% | 高度锁定，姿态变化小 | 表情微调、换背景 |

#### 3.4 奶盖的角色参考图集

同样为奶盖准备4个角度的参考图：

```
奶盖角色特征锁定清单：
- 品种：布偶猫，蓝眼睛，重点色（脸、耳朵、爪子深色）
- 标志配饰：粉色蝴蝶结领结
- 服装：奶油色/粉色针织衫
- 体型：圆润可爱，比豆包小一号
- 表情：傲娇但可爱，微微仰头
```

### 四、方法三：提示词锁定法

#### 4.1 标准化角色描述段落

为每个角色创建一段"角色描述模板"，每次生成时复制粘贴到提示词中：

**豆包角色描述模板**：
```
一只拟人化的金毛犬，金色蓬松毛发，大眼睛，温暖的笑容，
穿着蓝色卫衣和卡其色工装裤，戴着黑框眼镜，
体型健壮，耳朵自然下垂
```

**奶盖角色描述模板**：
```
一只拟人化的布偶猫，蓝眼睛，重点色毛发（脸和耳朵深棕色），
戴着粉色蝴蝶结领结，穿着奶油色针织衫，
体型娇小圆润，尾巴蓬松，微微仰头的傲娇表情
```

#### 4.2 提示词结构公式

```
[角色描述模板] + [动作/姿态] + [场景环境] + [光影/氛围] + [画质/比例]
```

**示例——豆包在播客场景**：
```
一只拟人化的金毛犬，金色蓬松毛发，大眼睛，温暖的笑容，
穿着蓝色卫衣和卡其色工装裤，戴着黑框眼镜，
坐在播客麦克风前，身体微微前倾，一只手搭在桌面上，
背景是温馨的播客工作室，暖色调灯光，
竖屏9:16比例，电影质感，柔和的演播室照明
```

**示例——奶盖在播客场景**：
```
一只拟人化的布偶猫，蓝眼睛，重点色毛发，
戴着粉色蝴蝶结领结，穿着奶油色针织衫，
坐在播客桌的另一侧，优雅地翘着二郎腿，一只手托着下巴，
背景是温馨的播客工作室，暖色调灯光，
竖屏9:16比例，电影质感，柔和的演播室照明
```

#### 4.3 关键一致性锚点词

在每张图的提示词中**必须包含**的角色锚点：

| 角色 | 必须包含的锚点词 | 作用 |
|------|-----------------|------|
| 豆包 | 金毛犬、金色蓬松毛发、蓝色卫衣、黑框眼镜 | 锁定物种+配色+配饰 |
| 奶盖 | 布偶猫、蓝眼睛、重点色、粉色蝴蝶结、奶油色针织衫 | 锁定物种+特征+配饰 |

### 五、三种方法的组合使用

#### 5.1 最佳实践：三法合一

| 步骤 | 方法 | 操作 |
|------|------|------|
| 1 | 参考图法 | 上传角色基准图作为参考 |
| 2 | 提示词法 | 在提示词中包含角色描述模板 |
| 3 | 种子法 | 固定使用已验证的种子值 |

**同时使用三种方法，角色一致性可达90%以上。**

#### 5.2 实战操作流程

```
1. 生成豆包的基准图（4个角度）→ 记录种子值
2. 将正面基准图作为参考图保存
3. 创建角色描述模板
4. 生成新场景时：
   a. 上传参考图（参考强度75%）
   b. 粘贴角色描述模板
   c. 填入种子值
   d. 添加场景描述
5. 生成4张，选择最佳
6. 如果满意，更新参考图为新图（保持一致性的同时积累更多角度）
```

### 六、常见问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 金毛犬变成了拉布拉多 | 品种描述不够精确 | 增加"金色蓬松长毛""耳朵下垂"等品种特征词 |
| 眼镜消失了 | 配饰在提示词中权重不够 | 用括号强调：`(戴着黑框眼镜)` |
| 拟人化程度不稳定 | 提示词中拟人描述不统一 | 统一使用"拟人化站立姿态""穿着人类服装"等描述 |
| 不同角度差异大 | 参考图只有正面 | 补充3/4侧面和半身参考图 |
| 换场景后角色变了 | 场景描述覆盖了角色描述 | 将角色描述放在提示词最前面 |

4. 能在场景母版图基础上快速生成同一场景的不同镜头

---

### 一、什么是场景资产库

#### 1.1 概念

场景资产库是一组预先生成好的"场景母版图"，包含你漫剧中所有常用场景的基准画面。后续生成具体镜头时，以母版图为参考，确保同一场景在不同镜头中风格、色调、布局完全一致。

#### 1.2 为什么宠物拟人漫剧需要场景资产库

| 问题 | 没有场景库 | 有场景库 |
|------|-----------|---------|
| 播客场景 | 每次生成的背景都不同 | 所有播客镜头背景一致 |
| 色调一致性 | 时冷时暖 | 统一色调 |
| 工作效率 | 每次重新描述场景 | 复用母版图，只改角色动作 |
| 镜头连贯性 | 场景跳变突兀 | 场景过渡自然 |

### 二、播客场景资产搭建（核心场景）

#### 2.1 播客工作室母版图生成

**Step 1：确定播客工作室的设计要素**

| 要素 | 豆包和奶盖的播客工作室设定 |
|------|-------------------------|
| 风格 | 温馨治愈系，有点日式简约 |
| 家具 | 木质播客桌，两把舒适的椅子 |
| 设备 | 两个专业麦克风，监听耳机，声卡 |
| 装饰 | 绿植、暖色台灯、照片墙 |
| 色调 | 暖棕色为主，点缀绿色和米白 |
| 灯光 | 暖色主灯+氛围灯带 |

**Step 2：生成全景母版图**

提示词：
```
一个温馨的播客工作室全景，木质桌面摆放着两个专业麦克风和监听耳机，
背景是简约的书架和绿植装饰，暖色调灯光，奶油色墙壁，
柔和的氛围灯带，竖屏9:16比例，电影质感，日式治愈风格
```

生成4张，选择最佳作为全景母版图。命名为：`播客工作室-全景-母版.png`

**Step 3：生成不同机位的母版图**

| 机位 | 提示词要点 | 命名 |
|------|-----------|------|
| 全景 | 桌面+两人座位+背景 | 播客工作室-全景-母版.png |
| 中景（豆包侧） | 豆包座位一侧的桌面和麦克风 | 播客工作室-豆包中景-母版.png |
| 中景（奶盖侧） | 奶盖座位一侧的桌面和麦克风 | 播客工作室-奶盖中景-母版.png |
| 桌面特写 | 桌面上的麦克风、耳机、杯子 | 播客工作室-桌面特写-母版.png |
| 背景特写 | 书架、绿植、照片墙 | 播客工作室-背景-母版.png |

#### 2.2 使用母版图生成具体镜头

**方法：以图生图**

1. 选择对应的母版图作为参考
2. 上传到即梦AI的"以图生图"功能
3. 输入角色和动作描述：
```
[豆包角色描述模板]，坐在播客桌左侧，身体前倾对着麦克风说话，
表情兴奋，[奶盖角色描述模板]坐在右侧，歪头认真倾听
```
4. 参考强度设为60-70%（保留场景，允许角色变化）

### 五、场景母版图的复用技巧

#### 5.1 同一场景不同时间/天气

以播客工作室为例，通过修改提示词生成不同氛围的变体：

| 氛围 | 提示词追加 | 用途 |
|------|-----------|------|
| 日常温暖 | "暖色调灯光，温馨氛围" | 日常播客 |
| 深夜录制 | "只有台灯亮着，窗外是夜色" | 深夜谈心 |
| 节日特别 | "装饰了彩灯和气球" | 节日特辑 |
| 阴天忧郁 | "阴天的自然光从窗户照入，色调偏冷" | 剧情转折 |

**操作方法**：将母版图作为参考，追加氛围描述，参考强度60-70%。

#### 5.2 场景转换的过渡帧

在漫剧中从一个场景切换到另一个场景时，需要生成"过渡帧"：

```
方法：
1. 截取当前场景最后一帧
2. 将下一场景的母版图作为结束帧
3. 在剪映中使用转场效果（淡入淡出、模糊过渡等）
```

### 六、常见问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 同一场景不同镜头差异大 | 没有使用母版图参考 | 每次都以母版图为参考生成 |
| 场景细节太多，角色被淹没 | 场景描述过于复杂 | 简化场景描述，突出角色区域 |

| 场景和角色风格不统一 | 分别生成时风格词不一致 | 统一使用相同的风格关键词 |

### 一、为什么要切换画风

#### 1.1 漫剧中的画风需求

| 场景 | 适合的画风 | 原因 |
|------|-----------|------|
| 日常播客 | 温暖写实动漫 | 亲切感、日常感 |
| 回忆场景 | 水彩/手绘风 | 怀旧、温暖 |
| 搞笑吐槽 | Q版/卡通 | 夸张、可爱 |
| 深夜谈心 | 电影写实质感 | 沉重、真实 |
| 梦境/幻想 | 梦幻插画风 | 超现实、唯美 |
| 片头片尾 | 简约平面设计 | 品牌感、高级感 |

#### 1.2 画风切换的核心挑战

在切换画风时，最大的问题是：**角色特征可能在新画风中丢失**。

比如豆包在写实动漫风中戴着眼镜很清晰，但切换到水彩风后眼镜可能变得模糊或消失。

### 三、保持角色一致性的画风切换方法

#### 3.1 方法一：参考图+画风关键词

**操作步骤**：
1. 准备豆包的写实动漫风基准图
2. 上传到即梦AI的"以图生图"
3. 在提示词中保留角色描述，替换画风关键词
4. 参考强度设为65-75%

**关键**：角色描述模板不变，只修改最后的画风词。

#### 3.2 方法二：逐步过渡法

如果直接从写实动漫跳到水彩差异太大，可以分步过渡：

```
写实动漫 → 半写实插画 → 水彩插画

Step 1：写实动漫基准图
Step 2：以基准图为参考，提示词改为"半写实插画风格，柔和色彩"
Step 3：以Step 2的结果为参考，提示词改为"水彩插画风格"
```

#### 3.3 方法三：角色锚点强化法

在切换画风时，在提示词中**加强角色特征描述的权重**：

```
// 普通描述
戴着黑框眼镜

// 强化描述
(戴着黑框眼镜:1.3)，(蓝色卫衣:1.2)，(金色蓬松毛发:1.2)
```

**注意**：即梦AI的权重语法可能因版本不同而有所差异，请以实际操作为准。

### 三、批量生产效率优化

#### 3.1 提示词模板化

为每种类型的生成建立标准模板：

**角色+表情+动作+场景模板**：
```
[角色描述模板]，[表情描述]，[动作描述]，
[场景描述]，[灯光描述]，竖屏9:16，[画风关键词]
```

**使用方式**：只需要替换方括号中的内容，其他保持不变。

#### 3.2 批量生成策略

| 策略 | 说明 | 效率提升 |
|------|------|---------|
| 同角色批量 | 一次生成同一角色的多个表情/动作 | 减少切换成本 |
| 同场景批量 | 一次生成同一场景的多个镜头 | 场景参考图复用 |
| 种子值复用 | 验证好的种子值反复使用 | 减少试错 |
| 参考图链式 | 上一张满意的结果作为下一张的参考 | 累积一致性 |

#### 3.3 质量控制检查点

每生成5-10张后，进行一次质量检查：

```
□ 角色一致性检查：所有图片中的豆包/奶盖是否像同一个人？
□ 场景一致性检查：同一场景的不同镜头是否风格统一？
□ 色调一致性检查：整体色调是否统一？
□ 配饰完整性检查：眼镜、蝴蝶结等配饰是否都在？
□ 画风一致性检查：所有图片是否使用同一画风？
```

## 模块五总结

### 核心知识点回顾

1. **角色一致性**：三法合一（种子锁定+参考图喂养+提示词锁定），确保豆包和奶盖永远"不换脸"

4. **表情与动作资产**：建立标准化的表情和动作库，覆盖播客对话的所有情绪需求
5. **生产流水线**：将角色、场景、表情、动作整合为高效的批量生产流程
