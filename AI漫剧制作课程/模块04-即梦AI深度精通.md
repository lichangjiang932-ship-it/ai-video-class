### 一、即梦AI是什么

即梦AI（Dreamina）是字节跳动推出的AI创作平台，是制作宠物拟人漫剧的主力工具。

**核心能力**：

| 功能 | 说明 | 在漫剧中的用途 |
|------|------|---------------|
| 文生图 | 输入文字描述→生成图片 | 生成角色立绘、场景母版图、首尾帧画面 |
| 图生图 | 上传图片+文字→生成新图 | 风格迁移、画面精修、换表情/换背景 |
| 文生视频 | 输入文字描述→生成视频 | 生成15秒以内的动态视频片段 |
| 图生视频 | 上传图片→让它"动起来" | 把静态首尾帧变成动态视频 |
| 角色参考 | 上传角色图→锁定外貌 | 保持豆包/奶盖跨镜头形象一致 |
| 局部重绘 | 涂抹区域→只修改该部分 | 修复手部变形、修改表情等 |

**为什么选即梦AI？**

- **免费额度充足**：每天有大量免费生成次数
- **中文理解强**：中文提示词效果优于海外工具
- **视频生成**：支持最长15秒的视频生成
- **角色一致性**：内置"角色参考"功能
- **与剪映打通**：生成内容可直接导入剪映剪辑

### 四、首次体验：端到端生成一个镜头

让我们用10分钟完成一次完整体验：

**Step 1：生成角色图片（文生图）**
1. 点击首页"文生图"
2. 输入提示词：
   ```
   一个阳光帅气的年轻男性，金色短发微卷，暖棕色眼睛，
   穿着橙色卫衣，开心地笑着说话，坐在播客录音间桌前，
   面前有专业麦克风，暖黄色灯光，中景正面镜头，
   日系动漫风格，高质量
   ```
3. 比例选择 9:16（竖屏）
4. 生成数量选择 4
5. 点击"生成"，等待15-30秒
6. 从4张结果中选择最佳的一张

**Step 2：让图片动起来（图生视频）**
1. 点击选中图片下方的"生成视频"按钮
2. 在运镜描述中输入：`角色微笑着说话，嘴巴微动，轻轻点头`
3. 点击"生成"，等待视频生成
4. 预览视频效果

**Step 3：下载使用**
1. 点击视频右上角的"下载"按钮
2. 视频保存到本地（MP4格式）
3. 可直接导入剪映进行后续剪辑

### 一、提示词编写规范

#### 1.1 提示词结构

```
[角色描述] + [动作表情] + [场景环境] + [镜头构图] + [风格质量]
```

**权重规则**：越靠前的词权重越高。角色和动作放最前面。

#### 1.2 从模糊到精确的优化过程

**第1版（太模糊）**：
```
一个男的在说话
```
问题：AI会随机生成任何人。

**第2版（加入角色）**：
```
一个金色短发的男的在说话
```
进步：有了基本特征，但不够具体。

**第3版（加入细节）**：
```
一个金色短发微卷的年轻男性，穿着橙色卫衣，在播客录音间说话
```
进步：有了服装和场景，但缺少情绪和构图。

**第4版（完整版）**：
```
一个阳光帅气的年轻男性，金色短发微卷，暖棕色眼睛，
穿着橙色卫衣，开心地笑着说话，嘴巴张开，
坐在播客录音间桌前，面前有专业麦克风，
暖黄色灯光，中景正面镜头，日系动漫风格，高质量
```
效果：角色、动作、场景、光线、构图、风格全部到位。

### 二、参数设置详解

#### 2.1 图片比例

| 比例 | 适用场景 | 推荐用途 |
|------|---------|---------|
| 9:16 | 竖屏，手机全屏 | 抖音/快手竖屏漫剧 |
| 16:9 | 宽屏横屏 | B站横屏番剧 |
| 1:1 | 方形 | 角色立绘、封面图 |
| 3:4 | 竖屏但不极端 | 漫画分格 |

**宠物播客漫剧推荐**：**9:16**（竖屏，适合抖音）

#### 2.2 模型选择

| 模型 | 特点 | 适用场景 |
|------|------|---------|
| 通用v2.0 | 质量好，细节丰富 | 正式生成，推荐日常使用 |
| 动漫专用模型 | 针对动漫风格优化 | 日漫/国漫风格漫剧 |
| 写实模型 | 照片级真实感 | 写实风格短剧 |

**宠物拟人漫剧推荐**：**动漫专用模型**或**通用v2.0**

#### 2.3 生成数量

- 可选1/2/4张
- **建议选4张**：同一提示词生成4张，从中挑选最佳
- 这是"抽卡"策略——多生成几张能大幅提高满意度

## 第20课：角色一致性技术——让豆包和奶盖"始终如一"

### 学习目标

1. 理解角色一致性的核心挑战
2. 掌握即梦AI"角色参考"功能的完整使用步骤
3. 学会多方案组合确保角色跨镜头形象稳定
4. 能独立为豆包和奶盖建立一致性工作流

### 一、为什么角色一致性是最大挑战

**问题本质**：AI每次生成都是随机过程，即使提示词完全相同，每次的角色外貌都会有差异。

**在播客漫剧中意味着什么**：
- 第1个镜头的豆包是圆脸
- 第5个镜头的豆包变成了长脸
- 观众会觉得"这不是同一个人"

**解决方案**：

| 方案 | 效果 | 操作难度 |
|------|------|---------|
| 角色参考功能 | ★★★★★ | 简单 |
| 固定提示词模板 | ★★★☆☆ | 简单 |
| 图生图接力 | ★★★★☆ | 中等 |
| 多方案组合 | ★★★★★ | 较复杂 |

### 二、角色参考功能使用步骤（核心方法）

#### Step 1：创建角色基准图

用文生图生成高质量的角色基准图：
- 面部清晰、姿态端正、光线均匀、背景简洁
- 正面或3/4侧面
- 选择最满意的一张作为基准图

#### Step 2：开启角色参考

1. 进入文生图界面
2. 在左侧找到"角色参考"开关
3. 点击开启

#### Step 3：上传基准图

1. 点击"上传参考图"
2. 选择豆包或奶盖的基准图
3. 等待上传完成

#### Step 4：设置参考强度

- **建议设置：0.7-0.85**
- 0.5以下：参考效果弱，角色可能不够像
- 0.7-0.85：保持特征的同时允许姿态变化（推荐）
- 0.9以上：非常像，但姿态可能僵硬

#### Step 5：输入提示词

保持角色外貌描述与基准图一致，只修改动作和表情：

```
[角色固定描述]，[新表情/动作]，[场景]，[镜头]，[风格]
```

#### Step 6：生成并验证

- 生成4张，检查角色是否与基准图一致
- 如果不够像，提高参考强度重试

### 三、多镜头一致性工作流

**标准工作流**：

```
角色基准图（1张）
    │
    ├── 镜头1：开心说话 → 生成4张，选1张
    ├── 镜头2：认真倾听 → 生成4张，选1张
    ├── 镜头3：委屈巴巴 → 生成4张，选1张
    ├── 镜头4：犯傻懵圈 → 生成4张，选1张
    └── 镜头5：感动落泪 → 生成4张，选1张
```

每个镜头都使用相同的基准图作为角色参考。

### 四、图生图接力法

**原理**：用上一个镜头的结果作为下一个镜头的参考底图。

**流程**：
```
镜头1（文生图+角色参考）→ 满意的图片A
    ↓
镜头2（图生图，上传A，重绘强度0.4-0.5）→ 满意的图片B
    ↓
镜头3（图生图，上传B，重绘强度0.4-0.5）→ 满意的图片C
```

**优点**：一致性极高
**缺点**：中间出问题会影响后续；风格可能逐渐偏移

**最佳实践**：每隔3-5个镜头，用角色基准图"校准"一次。

### 五、服装锁定技巧

**问题**：AI有时会改变角色服装。

**解决方案**：
1. 在提示词中极其详细地描述服装
2. 在角色参考中使用包含该服装的基准图
3. 如果还是变了，用图生图+低重绘强度（0.3）修复

### 六、常见问题

**Q1：角色参考强度设为最高（1.0）效果最好吗？**
> 不一定。过高会导致姿态僵硬。0.7-0.85通常是最优范围。

**Q2：参考图可以是真人照片吗？**
> 可以，但效果不如动漫风格的参考图。建议先转为动漫风格再使用。

**Q3：如何管理两个角色的参考图？**
> 建立"角色资产文件夹"，每个角色一个子文件夹，包含基准图和各角度参考图。

### 七、课后练习

1. **创建基准图**：为豆包和奶盖各创建一张高质量基准图
2. **角色参考测试**：用角色参考功能生成同一角色5个不同表情
3. **一致性对比**：对比"仅提示词"和"提示词+角色参考"的效果差异

## 第21课：文生视频——用文字描述生成15秒动态视频

### 学习目标

1. 理解即梦AI文生视频的功能和限制
2. 掌握视频提示词的编写方法
3. 学会生成适合播客场景的动态视频
4. 能独立为任意镜头生成15秒视频

### 一、文生视频功能概述

**什么是文生视频？**

输入文字描述，即梦AI生成对应的视频片段。

**关键限制**：
- 视频时长：通常4-15秒（取决于模型版本和设置）
- 分辨率：720p或1080p
- 帧率：24fps或30fps
- 内容复杂度：越简单的描述，生成效果越稳定

**在播客漫剧中的用途**：
- 生成角色说话的动态视频
- 生成角色表情变化的视频
- 生成环境空镜的动态视频
- 生成转场动画

### 二、视频提示词编写

#### 2.1 视频提示词 vs 图片提示词

| 维度 | 图片提示词 | 视频提示词 |
|------|-----------|-----------|
| 描述对象 | 静态画面 | 动态过程 |
| 重点 | 角色+场景+构图 | 动作+变化+时间 |
| 长度 | 100-150字 | 50-100字（更简洁） |
| 运动描述 | 不需要 | 必须包含 |

#### 2.2 视频提示词结构

```
[角色描述] + [动作/变化过程] + [场景] + [镜头运动] + [风格]
```

**关键区别**：视频提示词要描述"过程"而不是"状态"。

#### 2.3 播客场景视频提示词示例

**豆包说话**：
```
一个金色短发穿橙色卫衣的年轻男性，坐在播客录音间桌前，
微笑着说话，嘴巴一张一合，双手轻轻比划，
偶尔点头，镜头固定，日系动漫风格
```

**奶盖吐槽**：
```
一个银白色长发穿白色连衣裙的年轻女性，坐在播客录音间桌前，
听到什么后无语地叹气，单手扶额，微微摇头，
镜头固定，日系动漫风格
```

**豆包犯傻**：
```
一个金色短发穿橙色卫衣的年轻男性，坐在播客录音间桌前，
一脸懵的表情，眼睛慢慢瞪大，嘴巴慢慢张开，
头微微歪向一边，镜头缓慢推进，日系动漫风格
```

**奶盖微笑**：
```
一个银白色长发穿白色连衣裙的年轻女性，坐在播客录音间桌前，
嘴角慢慢上扬，露出温柔的微笑，眼神变得柔和，
镜头固定，日系动漫风格
```

**双人对话**：
```
两个年轻男女坐在播客录音间桌前，面对面交谈，
左侧金色短发男性在说话，右侧银白色长发女性在倾听，
偶尔点头回应，镜头缓慢左移，日系动漫风格
```

### 三、参数设置

#### 3.1 视频比例

与图片比例保持一致：
- 竖屏漫剧 → **9:16**
- 横屏漫剧 → **16:9**

#### 3.2 视频时长

- 短视频（4-6秒）：适合单句台词、表情变化
- 中视频（7-10秒）：适合2-3轮对话
- 长视频（11-15秒）：适合完整对话段落

**建议**：先用短视频测试效果，满意后再生成更长的版本。

#### 3.3 运镜描述

| 运镜方式 | 提示词 | 适用场景 |
|---------|--------|---------|
| 固定 | 镜头固定，镜头不动 | 常规对话 |
| 缓慢推进 | 镜头缓慢推进 | 强调表情、情绪升温 |
| 缓慢左移 | 镜头缓慢左移 | 从一个角色移到另一个 |
| 缓慢拉远 | 镜头缓慢拉远 | 揭示环境、收尾 |

### 四、生成流程

**Step 1：编写视频提示词**
- 按模板编写
- 保持简洁（50-100字）

**Step 2：设置参数**
- 比例：9:16
- 时长：根据需要选择

**Step 3：点击生成**
- 等待生成（通常1-5分钟）

**Step 4：预览和下载**
- 预览效果
- 满意则下载（MP4格式）
- 不满意则调整提示词重新生成

### 五、常见问题

**Q1：生成的视频时长不到15秒怎么办？**
> 即梦AI会根据内容复杂度自动决定时长。如果需要更长的视频，可以减少提示词中的动作描述，让AI有更多时间完成单一动作。

**Q2：视频中角色变形了怎么办？**
> 降低动作复杂度。描述越简单，AI越不容易出错。避免"同时做多个动作"的描述。

**Q3：视频的画面质量不如静态图怎么办？**
> 这是正常的。视频生成的计算量更大，画质会有损失。可以用"图生视频"功能（下一课讲解），基于高质量静态图生成视频，画质会更好。

2. **进阶练习**：生成奶盖吐槽的视频，要求包含"叹气+摇头"的动作
3. **挑战练习**：生成一个双人对话的15秒视频

## 第22课：图生视频——把静态关键帧变成动态画面

### 学习目标

1. 理解图生视频与文生视频的区别
2. 掌握图生视频的操作流程
3. 学会基于静态首尾帧生成动态视频
4. 能独立完成从静态图到动态视频的转化

### 一、图生视频 vs 文生视频

| 维度 | 文生视频 | 图生视频 |
|------|---------|---------|
| 输入 | 文字描述 | 静态图片+文字描述 |
| 画质 | 一般 | 更高（基于已有图片） |
| 可控性 | 较低 | 更高（首帧已确定） |
| 适用场景 | 快速测试 | 正式制作 |
| 角色一致性 | 较难控制 | 更容易保持（基于角色图） |

**在播客漫剧制作中，推荐使用图生视频**：
1. 先用文生图生成高质量的静态关键帧
2. 再用图生视频让关键帧"动起来"
3. 这样画质更好，角色一致性更高

### 二、图生视频操作流程

#### Step 1：准备静态关键帧

用文生图生成一张高质量的静态画面：
```
一个阳光帅气的年轻男性，金色短发微卷，暖棕色眼睛，
穿着橙色卫衣，开心地笑着说话，嘴巴张开，
坐在播客录音间桌前，面前有专业麦克风，
暖黄色灯光，中景正面镜头，日系动漫风格，高质量
```

选择最满意的一张，下载保存。

#### Step 2：进入图生视频

1. 在即梦AI首页点击"图生视频"
2. 上传准备好的静态关键帧

#### Step 3：编写运镜描述

运镜描述告诉AI"图片中的内容应该怎么动"：

**豆包说话**：
```
角色微笑着说话，嘴巴一张一合，偶尔点头，
双手轻轻比划，镜头固定
```

**奶盖吐槽**：
```
角色听到什么后叹气，微微摇头，
嘴角微微抽搐，镜头固定
```

**豆包犯傻**：
```
角色表情从正常慢慢变成懵圈，
眼睛逐渐瞪大，嘴巴慢慢张开，镜头缓慢推进
```

**奶盖微笑**：
```
角色嘴角慢慢上扬，露出温柔的微笑，
眼神变得柔和，镜头固定
```

**双人对话**：
```
左侧角色在说话，嘴巴微动，
右侧角色倾听，偶尔点头，镜头缓慢左移
```

#### Step 4：设置参数

- 比例：与原图一致（9:16）
- 时长：根据需要选择（建议先试5-8秒）

#### Step 5：生成并预览

- 点击"生成"
- 等待生成完成
- 预览效果

#### Step 6：下载使用

- 满意则下载（MP4格式）
- 不满意则调整运镜描述重新生成

### 三、首尾帧控制技巧

**核心概念**：在图生视频中，你可以通过指定"首帧"和"尾帧"来精确控制视频的起止状态。

#### 3.1 基本流程

```
Step 1：生成首帧静态图（文生图）
Step 2：生成尾帧静态图（文生图）
Step 3：进入图生视频，上传首帧
Step 4：在运镜描述中描述从首帧到尾帧的变化过程
Step 5：生成视频
```

#### 3.2 首尾帧设计示例

**示例：豆包从微笑变成大笑**

首帧提示词（文生图）：
```
一个金色短发穿橙色卫衣的年轻男性，微笑的表情，
嘴巴微张，坐在播客录音间桌前，
日系动漫风格，高质量
```

尾帧提示词（文生图）：
```
一个金色短发穿橙色卫衣的年轻男性，开心地大笑，
眼睛弯成月牙，露出洁白的牙齿，坐在播客录音间桌前，
日系动漫风格，高质量
```

运镜描述（图生视频）：
```
角色从微笑逐渐变成大笑，眼睛慢慢弯起来，
嘴巴张开露出笑容，表情越来越开心
```

#### 3.3 首尾帧衔接要点

- **主体一致**：首帧和尾帧中的角色必须是同一个人
- **位置一致**：角色在画面中的位置保持不变
- **背景一致**：同一场景的首尾帧背景相同
- **差异适度**：只改变表情或动作，其他不变

### 四、播客场景的图生视频工作流

**标准工作流**：

```
1. 用文生图生成每个镜头的首帧（静态图）
2. 用文生图生成每个镜头的尾帧（静态图）
3. 用图生视频，上传首帧，描述到尾帧的变化
4. 生成15秒视频
5. 下载保存
6. 导入剪映拼接
```

**效率技巧**：
- 同一镜头的首尾帧80%相同，只有表情部分不同
- 可以先生成首帧，然后用图生图+低重绘强度（0.3）微调表情得到尾帧
- 批量生成时，固定风格描述词和参数，只替换表情/动作

### 五、常见问题

**Q1：图生视频和文生视频应该先用哪个？**
> 正式制作用图生视频（画质好、可控性强）。快速测试用文生视频（速度快）。

**Q2：首尾帧的差异应该多大？**
> 差异太小→视频像静止的；差异太大→过渡不自然。建议差异控制在"一个表情变化"或"一个动作"的范围内。

**Q3：生成的视频有闪烁/变形怎么办？**
> 降低首尾帧的差异，或减少运镜描述中的动作数量。描述越简单，AI越稳定。

### 六、课后练习

1. **基础练习**：用图生视频将一张豆包静态图变成说话的动态视频
2. **进阶练习**：设计一对首尾帧（微笑→大笑），用图生视频生成过渡
3. **挑战练习**：为一个完整的对话段落（4轮对话）生成4段图生视频

## 第23课：即梦AI场景资产库——播客录音间的母版图与变体

#### 2.1 标准版母版图

```
一个温馨的播客录音间内部全景，木质桌面，

背景是木质书架和绿色植物，墙上挂着节目logo海报，
暖黄色灯光，舒适的沙发椅，无人，纯场景画面，
日系动漫风格，高质量场景设计，16:9比例
```

#### 2.2 母版图精修

**窗口视角**：
- 重绘强度：0.5-0.6
- 提示词：`从播客录音间窗户往外看，窗台上有小植物，窗外是城市天际线`

## 第24课：图生图精修——修复AI缺陷与画面优化

### 学习目标

1. 掌握图生图的核心参数——重绘强度
2. 学会修复AI常见缺陷（手部变形、表情不对等）
3. 掌握画面精修的迭代方法
4. 能独立处理漫剧制作中的画面问题

### 一、重绘强度详解

重绘强度是图生图最关键的参数：

| 重绘强度 | 效果 | 适用场景 |
|---------|------|---------|
| 0.1-0.3 | 轻微修改，保留原图大部分 | 微调色调、修复小瑕疵 |
| 0.4-0.6 | 中等修改，保留构图改变细节 | 风格迁移、换表情 |
| 0.7-0.8 | 较大修改，保留大致轮廓 | 大幅风格转换 |
| 0.9-1.0 | 几乎完全重绘 | 从草图生成精细画面 |

### 二、修复AI常见缺陷

#### 2.1 手部变形修复

**问题**：AI生成的手指数量不对或形状奇怪。

**修复方法**：
1. 上传有问题的图片到图生图
2. 重绘强度：0.3-0.4
3. 提示词中强调：`正常的人手，五根手指，自然的手部姿势`
4. 负面提示词：`畸形手，多余手指，变形的手`
5. 生成后检查，不满意就微调重绘强度

**预防方法**：
- 避免手部特写镜头
- 在提示词中描述手部动作时尽量简单（如"双手放在桌上"而不是"双手比划复杂手势"）

#### 2.2 表情不对修复

**问题**：应该是微笑但生成了严肃脸。

**修复方法**：
1. 上传原图到图生图
2. 重绘强度：0.3-0.4
3. 提示词强调：`温柔的微笑表情，明亮的眼睛，愉悦的面部表情`
4. 生成后对比，变化太小就提高到0.5，变化太大就降低到0.25

#### 2.3 服装不一致修复

**问题**：豆包明明穿的是橙色卫衣，生成结果变成了蓝色T恤。

**修复方法**：
1. 上传原图到图生图
2. 重绘强度：0.3
3. 提示词强调：`穿着橙色卫衣，与原图相同的服装`
4. 生成后检查

### 三、画面精修迭代流程

**对于重要画面（如开场镜头、关键笑点镜头）**：

```
第1轮：文生图生成基础画面（选最接近预期的）
    ↓
第2轮：图生图调整表情（重绘强度0.3-0.4）
    ↓
第3轮：图生图微调细节（重绘强度0.2-0.3）
    ↓
第4轮：局部重绘修复具体问题（如手部）
```

每轮只解决一到两个问题，不要贪多。

### 四、局部重绘基础

**什么是局部重绘？**
只修改图片的某一部分，保持其余部分不变。

**操作步骤**：
1. 选择图片 → 点击"编辑"→"局部重绘"
2. 用画笔涂抹需要修改的区域
3. 输入重绘提示词（只描述被涂抹区域）
4. 设置重绘强度（0.7-0.9）
5. 生成并检查

**适用场景**：
- 修复手部变形
- 修改面部表情
- 更换服装颜色
- 添加/移除画面元素

### 五、常见问题

**Q1：图生图和文生图应该先用哪个？**
> 先用文生图找到接近预期的基础画面，再用图生图精修。

**Q2：重绘强度设多少最合适？**
> 没有万能值。微调用0.2-0.3，换表情用0.3-0.4，换背景用0.5-0.6。多试几次找感觉。

**Q3：精修后画面变模糊了怎么办？**
> 使用"超分辨率"功能（如果有），或在图生图提示词中加入"高清，精致细节"。

### 六、课后练习

1. **修复练习**：生成一张有手部缺陷的图片，用图生图修复
2. **表情修改**：将一张严肃表情的图片改成微笑
3. **迭代精修**：对一张图片进行3轮迭代精修，记录每轮的变化

## 第25课：批量生产工作流——高效生成一集漫剧的全部素材

### 学习目标

1. 掌握即梦AI批量生产的工作流程
2. 学会管理大量生成素材
3. 理解从即梦AI到剪映的完整链路
4. 能独立完成一集漫剧的全部素材生成

### 一、批量生产工作流总览

```
Step 1：准备阶段（30分钟）
├── 确认脚本已完成
├── 确认分镜表已完成
├── 确认角色基准图已创建
└── 确认场景母版图已创建

Step 2：首帧生成（60分钟）
├── 按分镜表逐镜头生成首帧
├── 使用角色参考保持一致性
├── 每个镜头生成4张，选最佳
└── 按编号保存到素材文件夹

Step 3：尾帧生成（40分钟）
├── 基于首帧用图生图微调表情/动作
├── 或用文生图重新生成尾帧
└── 按编号保存到素材文件夹

Step 4：视频生成（60分钟）
├── 用图生视频将首帧变成动态视频
├── 编写运镜描述
├── 生成并下载
└── 按编号保存到视频文件夹

Step 5：质量检查（30分钟）
├── 检查角色一致性
├── 检查场景一致性
├── 检查首尾帧衔接
└── 标记需要重新生成的镜头

Step 6：补生成（30分钟）
├── 重新生成不满意的镜头
├── 再次检查
└── 最终确认

总时间：约4-5小时（一集10-15分钟的漫剧）
```

### 二、素材管理规范

#### 2.1 文件夹结构

```
漫剧项目/
├── 角色资产/
│   ├── 豆包-基准图.png
│   ├── 奶盖-基准图.png
│   └── 角色描述文档.txt
├── 场景资产/
│   ├── 播客录音间-母版图.png
│   └── 场景描述文档.txt
├── 第1集/
│   ├── 分镜表.md
│   ├── 首帧/
│   │   ├── S01-首帧.png
│   │   ├── S02-首帧.png
│   │   └── ...
│   ├── 尾帧/
│   │   ├── S01-尾帧.png
│   │   ├── S02-尾帧.png
│   │   └── ...
│   ├── 视频/
│   │   ├── S01-视频.mp4
│   │   ├── S02-视频.mp4
│   │   └── ...
│   └── 脚本.md
└── 剪映项目/
```

#### 2.2 命名规范

```
首帧：[集数]-[镜头编号]-首帧.png
示例：EP01-S01-首帧.png

尾帧：[集数]-[镜头编号]-尾帧.png
示例：EP01-S01-尾帧.png

视频：[集数]-[镜头编号]-视频.mp4
示例：EP01-S01-视频.mp4
```

### 三、效率优化技巧

#### 3.1 提示词模板化

为每个角色和场景建立固定模板，每次只修改变量部分：

```
豆包模板：
[豆包固定描述]，[表情变量]，[动作变量]，
坐在播客录音间桌前，面前有专业麦克风，
暖黄色灯光，[镜头类型变量]，日系动漫风格，高质量
```

#### 3.2 批量生成策略

- 不是每个镜头都需要4张预览
- 简单镜头（如空镜）生成2张即可
- 复杂镜头（如双人同框）生成4张
- 关键镜头（如开场、笑点）可以生成8张

#### 3.3 种子值复用

找到满意的生成结果后，记录其种子值：
- 相似场景可以用相同种子值
- 微调提示词时固定种子可以保持整体构图

### 四、从即梦AI到剪映

**导出流程**：
1. 在即梦AI中下载所有视频片段（MP4格式）
2. 打开剪映，新建项目（9:16竖屏）
3. 导入所有视频片段
4. 按分镜表顺序排列
5. 添加配音、字幕、BGM
6. 导出成片

**注意事项**：
- 即梦AI生成的视频分辨率可能不同，在剪映中统一调整
- 视频时长可能不精确，需要在剪映中裁剪
- 相邻视频片段之间可以添加转场效果

### 五、常见问题

**Q1：一集漫剧需要生成多少素材？**
> 10-15分钟的播客漫剧，约25-40个镜头，需要25-40组首尾帧+25-40段视频。

**Q2：生成时间太长怎么办？**
> 熟练后可以大幅提速：建立资产库后，后续镜头只需复制粘贴+微调提示词。预计第二集开始，生成时间缩短50%。

**Q3：素材太多管理不过来怎么办？**
> 严格遵守命名规范和文件夹结构。每完成一个镜头就立即归档，不要堆到最后一起整理。

### 六、课后练习

1. **完整实践**：为一集漫剧生成全部首帧素材（至少15个镜头）
2. **效率记录**：记录每步的时间，找出最耗时的环节
3. **流程优化**：思考如何优化你的工作流，提高效率

- 尾帧图片：30-40张
- 视频片段：30-40段
- 预计总时间：4-6小时

### 二、制作流程

#### Day 1：准备阶段

**上午**：
1. 确认脚本已完成（模块二的成果）

3. 建立素材文件夹结构

**下午**：
1. 生成豆包角色基准图

**下午**：
1. 补生成不满意的镜头
2. 最终确认所有素材
3. 整理素材文件夹
