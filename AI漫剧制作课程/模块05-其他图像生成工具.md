### 一、通义万相概述

**通义万相（Tongyi Wanxiang）** 是阿里巴巴达摩院推出的 AI 图像生成大模型，于 2023 年上线，持续迭代。它具备以下核心能力：

| 能力 | 说明 |

| 图生图 | 上传参考图 + 文字指令，进行风格迁移或内容修改 |
| 人像写真 | 上传人脸照片，生成多种风格的写真 |
| 局部重绘 | 对图像指定区域进行重新生成 |
| 智能扩图 | 扩展图像边界，自动补全内容 |
| 视觉理解 | 对图像内容进行识别和描述 |

**访问方式**：

### 三、图生图功能详解

#### 3.1 功能说明

图生图允许你上传一张参考图片，并通过文字指令对其进行修改或风格转换。这在漫剧制作中极为实用：

- **风格转换**：将照片转为动漫风格
- **角色换装**：改变角色的服装、发型
- **场景替换**：保持角色，更换背景
- **表情调整**：修改角色的面部表情

#### 3.2 操作步骤（图生图）

**Step 1**：在通义万相中选择「图生图」标签页

**Step 2**：上传参考图片
- 点击上传区域，选择本地图片
- 支持 JPG、PNG 格式
- 建议分辨率不低于 512×512

**Step 3**：输入修改指令

```
将画面风格改为赛博朋克风格，背景替换为霓虹灯闪烁的未来城市街道，
角色保持不变，增加科技感的光效
```

**Step 4**：调整相似度滑块
- **相似度高（0.7-0.9）**：保留更多原图特征，适合微调
- **相似度低（0.3-0.5）**：大幅改变，适合风格转换

**Step 5**：点击「生成」，等待结果

**Step 6**：选择满意的版本下载

#### 3.3 漫剧场景应用实例

**场景：角色表情序列制作**

假设你需要为一个角色制作「惊讶→愤怒→悲伤」的表情序列：

1. 先用文生图生成角色的基准表情（平静）
2. 上传基准图，输入指令："表情变为惊讶，嘴巴微张，眼睛睁大"
3. 上传基准图，输入指令："表情变为愤怒，眉头紧锁，嘴角下沉"
4. 上传基准图，输入指令："表情变为悲伤，眼含泪水，嘴角下垂"

这样可以确保表情序列中角色的外貌特征保持一致。

### 四、常见问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 生成的手指数量不对 | AI 图像生成的通病 | 使用局部重绘功能，指定"正确的手，五根手指"重新生成手部 |
| 角色外貌不一致 | 每次生成的随机性 | 使用图生图，以上一次的结果作为参考 |
| 中文提示词效果不如英文 | 模型训练数据差异 | 尝试用英文重写关键描述，或中英混合 |
| 生成速度慢 | 服务器繁忙 | 避开高峰期，或使用 API 调用 |
| 图片分辨率不够 | 默认输出限制 | 使用「高清修复」功能进行超分辨率放大 |

### 二、图像生成功能

#### 2.1 操作步骤

**Step 1**：访问 https://hunyuan.tencent.com/ ，登录腾讯账号

**Step 2**：在对话框中输入图像生成指令

混元的特色是通过**对话式交互**生成图像，你可以像聊天一样描述需求：

```
请帮我生成一张图片：一个穿着古装的侠女，手持折扇，
站在月光下的竹林中，水墨画风格，意境悠远
```

**Step 3**：等待生成结果（通常 15-30 秒）

**Step 4**：查看生成的图片，如果不满意可以继续对话修改：

```
很好，但请把月亮画得更大一些，增加一些萤火虫的光点
```

**Step 5**：满意后点击图片下载

#### 2.2 对话式修改技巧

混元最大的特色是支持**多轮对话修改**，这对于漫剧制作非常实用：

```
第一轮："生成一个穿蓝色制服的男警察角色"
第二轮："把制服改为黑色西装，增加一条红色领带"
第三轮："表情改为严肃，背景换成办公室"
第四轮："增加一束从左侧照来的光线，制造戏剧性阴影"
```

每一轮修改都在前一次结果的基础上进行，保持角色基本特征不变。

### 三、数字人创作流程

#### 3.1 什么是 AI 数字人

AI 数字人是基于真人照片或视频，通过 AI 技术生成的虚拟人物形象。在漫剧制作中，数字人可以用于：

- **角色演绎**：让数字人"表演"漫剧中的角色
- **口播配音**：数字人根据台词自动对口型
- **虚拟主播**：用于漫剧的宣传和解说

#### 3.2 腾讯混元数字人创作步骤

**Step 1**：准备素材
- 一张清晰的正面人脸照片（分辨率建议 512×512 以上）
- 背景尽量简洁
- 光线均匀，无明显阴影

**Step 2**：进入数字人功能
- 在混元平台选择「数字人」或「智能体」相关功能
- 上传准备好的照片

**Step 3**：配置数字人参数

| 参数 | 说明 | 推荐设置 |
|------|------|----------|
| 形象风格 | 数字人的视觉风格 | 写实/卡通/动漫 |
| 声音类型 | 数字人的语音特征 | 根据角色选择 |
| 动作模板 | 预设的动作模式 | 根据场景选择 |

**Step 4**：输入台词/脚本
```
大家好，欢迎来到我们的漫剧世界。
今天我要给大家讲一个关于勇气和友情的故事……
```

**Step 5**：生成数字人视频
- 系统会自动对口型、生成表情动作
- 等待处理完成（根据时长可能需要几分钟）

**Step 6**：预览和下载
- 检查口型同步、表情自然度
- 下载视频文件用于后期编辑

#### 3.3 漫剧中的数字人应用场景

1. **片头片尾**：数字人角色进行自我介绍或预告
2. **旁白解说**：数字人作为"说书人"推动剧情
3. **角色互动**：不同数字人角色之间的对话场景
4. **宣传推广**：数字人角色在社交媒体上宣传漫剧

### 四、常见问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 对话式修改效果不理想 | 描述不够具体 | 提供更精确的修改指令，指定具体区域 |
| 数字人口型不同步 | 音频质量问题 | 使用清晰、无噪音的音频 |
| 生成图片风格不统一 | 多次对话累积偏差 | 定期重新开始对话，使用参考图锚定风格 |
| 数字人表情僵硬 | 限制于预设模板 | 尝试不同的动作模板组合 |

### 一、Kolors 概述

**Kolors（可图）** 是快手自研的文生图模型，于 2024 年开源。

**核心优势**：
- **开源免费**：模型权重完全开放，可本地部署
- **中文原生支持**：对中文提示词有出色的理解能力
- **高质量输出**：图像质量媲美商业模型
- **高度可定制**：支持 LoRA、ControlNet 等扩展
- **社区活跃**：持续有社区贡献的微调模型和工作流

**技术规格**：
- 基于 Latent Diffusion 架构
- 支持 1024×1024 原生分辨率
- 提供 Base 模型和 Inpainting 模型

**访问方式**：
- 在线体验：https://kolors.kuaishou.com/
- GitHub 仓库：https://github.com/Kwai-Kolors/Kolors
- ComfyUI 集成：通过自定义节点使用
- Hugging Face：可下载模型权重

### 一、LoRA 原理与价值

#### 1.1 什么是 LoRA

**LoRA（Low-Rank Adaptation）** 是一种高效的模型微调技术。简单来说：

- **完整微调**：需要修改整个模型的数百万个参数，需要海量数据和算力
- **LoRA 微调**：只训练一小部分"适配层"参数（通常几 MB），效果却接近完整微调

对于漫剧制作来说，LoRA 的价值在于：

| 优势 | 说明 |
|------|------|
| 角色一致性 | 训练后每次生成都是同一个角色，解决 AI "换脸"问题 |
| 风格统一 | 可以训练特定画风，保持整部漫剧风格一致 |
| 高效训练 | 30-50 张图即可训练，几小时完成 |
| 轻量存储 | 一个 LoRA 文件通常只有 10-100MB |
| 灵活叠加 | 可以同时加载多个 LoRA（角色+风格+场景） |

#### 1.2 LoRA 训练的基本流程

```
准备训练数据 → 标注图片 → 配置训练参数 → 开始训练 → 测试效果 → 迭代优化
     ↓              ↓            ↓            ↓          ↓          ↓
  30-50张图    自动生成标签    设置学习率等   等待数小时   在ComfyUI中   调整参数
  高质量筛选   手动校正       配置文件       监控loss     加载LoRA     重新训练
```

**方法二：照片拍摄法**

1. 让真人模特穿上角色服装

**Step 1：裁剪与缩放**

使用 Python 脚本批量处理：

```python
from PIL import Image
import os

def preprocess_images(input_dir, output_dir, size=512):
    os.makedirs(output_dir, exist_ok=True)
    for filename in os.listdir(input_dir):
        if filename.lower().endswith(('.png', '.jpg', '.jpeg')):
            img = Image.open(os.path.join(input_dir, filename))
            # 保持宽高比缩放
            img.thumbnail((size, size), Image.Resampling.LANCZOS)
            # 创建正方形画布
            new_img = Image.new('RGB', (size, size), (255, 255, 255))
            offset = ((size - img.width) // 2, (size - img.height) // 2)
            new_img.paste(img, offset)
            new_img.save(os.path.join(output_dir, filename))

preprocess_images('./raw_images', './processed_images', 512)
```

```python
from rembg import remove
from PIL import Image
import os

def remove_background(input_dir, output_dir):
    os.makedirs(output_dir, exist_ok=True)
    for filename in os.listdir(input_dir):
        if filename.lower().endswith(('.png', '.jpg', '.jpeg')):
            input_path = os.path.join(input_dir, filename)
            output_path = os.path.join(output_dir, filename.replace('.jpg', '.png'))
```

```toml
optimizer_type = "AdamW8bit"

# 保存与日志
save_every_n_epochs = 2   # 每 2 个 epoch 保存一次
log_prefix = "my_character"
logging_dir = "./training_data/log"
```

#### 4.2 参数详解与调优

| 参数 | 说明 | 推荐值 | 调优建议 |
|------|------|--------|----------|
| network_dim | LoRA 秩（rank） | 32-128 | 角色训练推荐 64，风格训练推荐 32 |
| network_alpha | 缩放因子 | dim/2 | 通常为 dim 的一半 |
| learning_rate | 学习率 | 1e-4 | 过大会过拟合，过小会欠拟合 |
| max_train_epochs | 训练轮数 | 10-20 | 30 张图推荐 10 轮，50 张图推荐 15 轮 |
| batch_size | 批次大小 | 1-4 | 显存够大可增加，通常 1 即可 |
| resolution | 训练分辨率 | 512/768/1024 | SD1.5 用 512，SDXL 用 1024 |

#### 4.3 防止过拟合

过拟合的表现：生成图片质量下降，出现伪影，色彩异常。

**防过拟合策略**：
1. **降低学习率**：从 1e-4 降到 5e-5
2. **减少训练轮数**：从 15 轮降到 8 轮
3. **增加数据量**：从 30 张增加到 50 张
4. **使用 dropout**：在配置中添加 `network_dropout = 0.1`
5. **使用正则化图片**：添加 10-20 张通用图片作为正则化数据

### 六、常见问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 训练后角色不像 | 数据质量差或数量不足 | 重新筛选数据，确保特征一致 |
| 过拟合（生成图有伪影） | 学习率太高或轮数太多 | 降低学习率，减少 epoch |
| 欠拟合（角色特征不明显） | 学习率太低或轮数不够 | 增加学习率，增加 epoch |
| 显存不足（OOM） | 图片太大或 batch 太大 | 降低分辨率到 512，batch_size=1 |
| 训练速度太慢 | GPU 性能不足 | 使用更小的 dim（32），减少数据量 |
| 生成图片模糊 | LoRA 强度太高 | 降低 strength 到 0.6-0.7 |
