### 7.4 文心5.0进阶技巧

#### 技巧1：多轮对话迭代法

```
第一轮：生成初稿
→ "请生成第1集剧本初稿"

第二轮：针对性优化
→ "第3场的对白太书面了，改成更口语化的版本，要有网感"

第三轮：细节打磨
→ "第5场的动作描写加上角色的微表情变化，要体现内心的纠结"

第四轮：最终确认
→ "整体检查一遍节奏，前3秒钩子够不够强？结尾悬念够不够？"
```

#### 技巧2：角色一致性维护法

在每集生成时，先发送角色"锚定提示词"：

```
【角色锚定信息】
主角：林辰，22岁，黑发短寸，剑眉星目，穿黑色卫衣，性格外冷内热

反派：赵天，28岁，油头西装，鹰钩鼻，眼神阴鸷，性格阴险狡诈

请在生成剧本时严格保持以上角色形象描述的一致性。
每涉及以上角色出场时，先用一句话描写外貌特征。
```

### 7.5 常见问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 生成的剧本太长，超出单集时长 | 没有限定字数 | 在提示词中明确"总字数控制在500字以内" |
| 角色前后描述不一致 | 上下文遗忘 | 每集开始前重新发送角色锚定信息 |
| 对白太书面，不像真人说话 | 模型默认风格 | 明确要求"口语化、有网感、像00后说话" |
| 剧情节奏拖沓 | 缺乏节奏约束 | 要求"每场必须推进剧情或制造冲突，删除无意义过渡" |
| 结尾没有钩子 | 未强调钩子机制 | 要求"最后一句必须是悬念句，格式：'[钩子] XXX？'" |
| 场景描述不够可视化 | 未针对AI制作 | 要求"每个场景描述必须包含：颜色、光线、构图、氛围" |
| 重复套路感强 | 缺乏创新指令 | 提供参考作品："参考《XX》的叙事方式，不要用常见套路" |

### 8.4 DeepSeek高级技巧

```
请从以下角度逐步分析：

4. 最后提出3个创新的差异化方向，每个方向给出具体的故事设定

请一步步来，每一步都要深入分析。
```

#### 技巧2：自我检查法

```
你刚刚生成的第X集剧本，请用以下维度进行自我检查：

4. 【AI适配检查】场景描述是否足够可视化？有没有AI难以生成的复杂场景？
5. 【商业检查】这集放在哪个位置？有没有设置付费卡点的潜力？
```


### 8.5 常见问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 联网搜索结果不准确 | 搜索关键词不够精准 | 用更具体的关键词，如"2026年4月 抖音漫剧 玄幻 爆款" |
| 生成的剧本有逻辑漏洞 | 没有利用推理能力 | 加入"请先检查逻辑再输出"的指令 |
| 对白风格不符合要求 | 未明确目标风格 | 提供示例对白："参考这种风格：'你谁啊？我凭什么信你？'" |
| 角色行为OOC | 上下文过长导致遗忘 | 每集开始前重新发送角色核心设定（精简版） |
| 剧本太长/太短 | 字数控制不精确 | 明确"总字数XXX字，其中对白占XX%，描写占XX%" |
| 付费卡点设置不当 | 缺乏商业思维 | 明确要求"在观众情绪最高点切断，制造强烈好奇心" |
| API调用超时 | 单次生成内容过多 | 分场次生成，最后合并 |

```
   - 说话习惯（口头禅、语气、语速）

   - 社交模式（对不同人的态度差异）

3. 【角色弧线】
   - 出发状态（故事开始时的样子）
   - 转变契机（什么事件改变了他/她）
   - 成长过程（如何一步步变化）
   - 最终状态（故事结束时的样子）

4. 【关系网络】
   - 与其他核心角色的关系
   - 关系中的权力动态
   - 关系的变化轨迹
   - 关键冲突点

5. 【标志性场景】
   - 请设计3个能体现该角色核心特质的标志性场景
   - 每个场景包含：场景描述、角色行为、对白、情绪

请输出完整的角色档案。
```

### 11.4 GLM-4进阶技巧

#### 技巧1：角色"压力测试"

```
我设计了一个角色：[角色描述]

请对这个角色进行"压力测试"——设计5个极端场景，
测试角色在极端情况下的反应：

1. 生死抉择场景：角色必须在两个重要的人之间选择救一个
2. 信仰崩塌场景：角色发现自己一直坚信的东西是假的
3. 道德困境场景：角色必须做一件违背自己原则的事才能达成目标
4. 孤独极限场景：角色被所有人背叛，孤立无援
5. 诱惑极限场景：角色面对巨大的诱惑，考验其底线

每个场景给出：
- 角色的可能反应（至少2种）
- 反应是否符合角色设定
- 如何在剧本中使用这个场景
```

#### 技巧2：角色"前后传"生成

```
请为角色[角色名]撰写前后传：

【前传】（故事开始前5年发生了什么）
- 角色当时的状态
- 发生了什么关键事件
- 这个事件如何塑造了角色

【后传】（故事结束后5年的发展）
- 角色最终的结局
- 角色关系的最终状态
- 角色的成长总结

前传和后传各500字左右。
```

### 11.5 常见问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 角色设定过于扁平 | 缺乏深度挖掘 | 使用"压力测试"找出角色的多面性 |
| 多轮对话后角色声音趋同 | 模型倾向标准化回复 | 每次对话前重新发送角色的"一句话标签"和"经典台词" |
| 角色关系设计过于简单 | 未考虑动态变化 | 要求设计"关系的五个阶段"而非静态关系 |
| 人格卡片缺乏实用性 | 格式太学术 | 要求"用写给人看的方式，不要写成论文" |
| 群戏中角色存在感不均 | 未明确分配功能 | 为每个角色设定"群戏定位"（推动者/吐槽者/和事佬等） |

4. **思考题**：GLM-4在角色塑造上和其他模型相比，最独特的优势是什么？

### 12.2 AI剧本创作方法论全景

```
Phase 1           Phase 2           Phase 3           Phase 4
热点抓取    →     选题决策    →     故事构建    →     剧本落地
│                 │                 │                 │
▼                 ▼                 ▼                 ▼
· 联网搜索        · 竞品分析        · 世界观设定      · 大纲生成

· 趋势研判        · 差异化定位      · 剧情架构        · 对白打磨
· 用户画像        · 可行性评估      · 爽点分布        · 钩子优化
```

### 12.3 Phase 1：热点抓取（用DeepSeek联网搜索）

#### Step 1：数据采集

**提示词模板——热点数据采集**：

```
请联网搜索以下数据，我需要全面了解当前漫剧市场：

【搜索任务】
1. "2026年4月 抖音漫剧 热门题材" → 获取题材趋势
2. "漫剧 爆款 完播率 数据" → 获取用户偏好数据
3. "短剧 玄幻/都市/悬疑/甜宠 播放量" → 获取品类数据
4. "漫剧 创作者 月收入 案例" → 获取商业数据
5. "AI漫剧 新趋势 新技术" → 获取行业动态

【输出要求】
整理为一个结构化报告，包含：
1. 当前TOP5热门题材（附播放量数据）
2. 用户偏好变化趋势
3. 竞争红海/蓝海判断
4. 值得关注的新机会点
```

#### Step 2：趋势研判

**提示词模板——趋势分析**：

```
基于以上数据，请分析以下趋势：

1. 【上升趋势】哪些题材正在变热？为什么？
2. 【下降趋势】哪些题材在降温？原因是什么？
3. 【蓝海机会】哪些题材竞争少但需求大？
4. 【创新方向】有哪些新的题材融合可能成为爆款？

每个趋势给出：
- 判断依据（数据支撑）
- 预判时间窗口（还能火多久）
- 建议策略（现在入场/观望/放弃）
```

### 12.4 Phase 2：选题决策

#### Step 1：竞品分析矩阵

**提示词模板——竞品矩阵分析**：

```
我打算做[题材方向]的漫剧。请帮我做竞品矩阵分析：

【搜索并分析】
1. 搜索该题材下TOP10漫剧作品
2. 分析每部作品的：
   - 核心卖点
   - 播放量/点赞量
   - 更新频率
   - 制作水平
   - 用户评价（评论区关键词）

【输出竞品矩阵】
用表格呈现，维度包括：
| 作品名 | 核心卖点 | 播放量 | 制作水平 | 差异化空间 |

【差异化建议】
基于竞品分析，给我3个差异化的切入方向。
```

#### Step 2：选题决策模型

**选题评分表**：

| 评估维度 | 权重 | 评分标准（1-10分） |
|----------|------|-------------------|
| 市场热度 | 20% | 搜索指数、相关作品播放量 |
| 竞争程度 | 20% | 同题材作品数量（越少分越高） |
| 差异化空间 | 20% | 是否有独特的切入角度 |
| 变现潜力 | 15% | 付费意愿、广告适配度 |
| 制作难度 | 15% | AI工具能否胜任、素材获取难度 |
| 个人兴趣 | 10% | 是否能持续创作不厌倦 |

**决策公式**：
```
总分 = Σ（维度得分 × 权重）
总分 ≥ 7分 → 强烈推荐
总分 5-7分 → 可以尝试
总分 < 5分 → 建议换方向
```

### 12.5 Phase 3：故事构建

#### Step 1：世界观设定

**提示词模板——世界观构建**：

```
请帮我构建漫剧《[剧名]》的世界观。

【基础设定】

- 核心规则：[这个世界的运行法则是什么]
- 力量体系：[如果有的话，能力如何分级]
- 社会结构：[阶层、组织、势力分布]

【请设计】
1. 世界观简介（200字，可作为旁白使用）
2. 世界的核心矛盾（驱动剧情的根本冲突）
3. 世界中的"常识"（角色们认为理所当然的事）
4. 世界中的"禁忌"（不能做的事，违反会怎样）
5. 世界中的"机会"（主角可以利用的规则漏洞）

要求：世界观要简洁有力，能在30秒内讲清楚。
```

#### Step 2：爽点设计系统

**爽点类型与设计方法**：

| 爽点类型 | 触发条件 | 情绪效果 | 适用场景 |
|----------|----------|----------|----------|
| 逆袭打脸 | 被看不起后证明自己 | 痛快、解气 | 主角受辱后反击 |
| 装逼打脸 | 低调实力者被轻视后展示实力 | 震惊、膜拜 | 主角隐藏身份被揭穿 |
| 复仇成功 | 长期压迫后终于反击 | 释放、痛快 | 反派被收拾 |
| 甜蜜互动 | 两情相悦的暧昧时刻 | 心动、上头 | CP互动场景 |
| 能力觉醒 | 危机时刻爆发新力量 | 燃、热血 | 战斗高潮 |
| 反转揭秘 | 发现隐藏的真相 | 震惊、恍然大悟 | 剧情转折点 |
| 团队集结 | 志同道合者汇聚 | 感动、燃 | 关键战役前 |
| 绝境翻盘 | 陷入绝境后逆转 | 刺激、兴奋 | 生死关头 |

**提示词模板——爽点分布设计**：

```
请为我设计24集漫剧的爽点分布图。

【基本信息】
- 题材：[题材]
- 核心爽点类型：[选择1-2种]
- 节奏偏好：[高燃型/波浪型/递增型]

【请输出】
1. 24集的情绪曲线（每集标注情绪类型和强度1-10）
2. 每3集一个小高潮，每6集一个大高潮
3. 标注每集的核心爽点类型
4. 标注付费卡点位置（最佳付费解锁集数）
5. 给出节奏建议（哪些集是高能集，哪些是过渡集）

用表格形式输出：
```

## 第13课：AI漫剧剧本特殊要求——可视化描写、镜头语言与AI生成适配

### 13.1 学习目标

- 理解AI漫剧剧本与传统影视剧本的本质区别
- 掌握"可视化描写"的写作技巧
- 学会用镜头语言设计每个画面
- 能写出完全适配AI生图/生视频的剧本

### 13.2 AI漫剧剧本 vs 传统影视剧本

| 维度 | 传统影视剧本 | AI漫剧剧本 |
|------|-------------|-----------|
| 画面控制 | 导演/摄影师二度创作 | 文字直接决定AI生成结果 |
| 描写自由度 | 可以写"美得不可方物" | 必须写"长发及腰，杏眼樱唇，白色连衣裙" |
| 镜头语言 | 专业导演理解 | 需要写得AI能理解 |
| 角色一致性 | 演员天然一致 | 靠文字描述维持一致 |
| 动作描写 | 可以含糊 | 必须精确到具体动作 |
| 场景描写 | 美术组补充 | 必须包含所有视觉元素 |
| 核心差异 | 写给"人"看 | 写给"AI+人"看 |

### 13.3 可视化描写技巧

#### 原则1：具体化——把抽象变成具象

```
❌ 错误示范（抽象描写）：
"她很美，站在那里，所有人都看呆了。"

✅ 正确示范（可视化描写）：
"苏晴站在路灯下，暖黄色的光勾勒出她纤细的侧影。
长发被夜风轻轻吹起，露出白皙的脖颈。
她微微侧头，杏眼在灯光下像琥珀一样透亮，
嘴角带着若有若无的笑意。
路过的人不自觉地放慢脚步，有人甚至撞上了电线杆。"
```

#### 原则2：画面化——用"摄影机视角"写

```
❌ 错误示范（纯叙事）：
"林辰走进办公室，发现自己的东西被收走了。"

✅ 正确示范（画面化）：
"镜头从办公室门口向内推进——
工位上空空荡荡，电脑显示器消失了，
只剩下一张光秃秃的桌面和一把歪斜的椅子。
桌角贴着的'林辰'工牌还在，
但名字上被人用红笔画了一个大大的叉。
林辰的手停在门把手上，指节发白。"
```

#### 原则3：五感化——调动视觉以外的感官

```
❌ 错误示范（只有视觉）：
"下雨了，林辰站在雨中。"

✅ 正确示范（多感官）：
"雨点砸在林辰的脸上，冰凉刺骨。
衬衫湿透了，贴在身上，勾勒出消瘦的轮廓。
雨水顺着发梢滴落，在下巴汇聚，然后坠落。
远处的霓虹灯在雨幕中化成一团团模糊的光晕，
像被打翻的颜料盘。"
```

### 13.4 镜头语言在剧本中的应用

#### 常用镜头类型与剧本标注

| 镜头类型 | 标注方式 | 适用场景 | AI生图提示词关键词 |
|----------|----------|----------|-------------------|
| 特写（CU） | 【特写】 | 表情变化、关键物品 | close-up, detail, face |
| 中景（MS） | 【中景】 | 对话、日常互动 | medium shot, waist up |
| 远景（LS） | 【远景】 | 环境展示、氛围营造 | wide shot, landscape |
| 全景（FS） | 【全景】 | 展示角色与环境关系 | full shot, environment |
| 俯拍 | 【俯拍】 | 压迫感、上帝视角 | overhead view, bird's eye |
| 仰拍 | 【仰拍】 | 威严感、力量感 | low angle, looking up |
| 过肩镜头 | 【过肩】 | 对话场景、窥视感 | over the shoulder |
| 跟拍 | 【跟拍】 | 追逐、运动场景 | tracking shot, following |
| 推镜头 | 【推镜头】 | 强调、紧张感 | push in, zoom in |
| 拉镜头 | 【拉镜头】 | 揭示环境、疏离感 | pull out, zoom out |

#### 镜头组合设计示例

```
SCENE 1 | 夜晚 · 城市天台 · 月光

【远景】城市夜景，万家灯火，镜头从高处俯瞰。
城市的声音远远传来，像白噪音一样模糊。
AI提示词：wide shot, city skyline at night, moonlight, cinematic

【中景】天台边缘，林辰的背影。他双手撑在栏杆上，
微微低头，肩膀线条紧绷。
AI提示词：medium shot, young man standing on rooftop, back view, night

【特写】林辰的手，指节发白，青筋暴起。
镜头缓缓上移——他的侧脸，眉头紧锁，
咬肌微微鼓起，眼神里有压抑的怒火。
AI提示词：close-up, male hand gripping railing, veins visible; 
transition to side profile, intense expression, moonlight

【过肩】从林辰肩膀后方拍过去，
面前的手机屏幕亮着——一条未读消息：
"你的代码，一文不值。——赵天"
AI提示词：over shoulder shot, phone screen glowing, 
text message visible, night atmosphere
```

### 13.5 AI生图/生视频适配要点

#### 要点1：避免复杂多人同框

```
❌ 不适合AI生成：
"七个人围坐在圆桌前开会，每个人表情各异。"

✅ 适合AI生成：
"林辰坐在会议桌前，对面是赵天。
两人之间隔着一份合同，气氛剑拔弩张。
其他人的背影虚化在画面两侧。"
（聚焦2人，其余用虚化处理）
```

#### 要点2：动作描写要"可执行"

```
❌ 不适合AI生成：
"他使出全力，施展了一套精妙绝伦的剑法。"

✅ 适合AI生成：
"林辰右手握剑，从右上方向左下方斜劈。
剑刃划破空气，带起一道银色弧光。
他的身体随剑势旋转半圈，
左脚为轴，右脚画弧，稳稳落地。"
（每个动作都是具体的、可执行的）
```

#### 要点3：场景描述要包含完整的视觉元素

```
❌ 不够完整的场景描述：
"他们来到了一个废弃的工厂。"

✅ 完整的场景描述：
"废弃的水泥工厂，墙壁斑驳，爬满了枯黄的藤蔓。
阳光从破损的屋顶洒下来，形成一道道光柱。
地上散落着生锈的铁链和碎玻璃。
空气中弥漫着铁锈和灰尘的味道。
角落里，一台巨大的机器静默着，
齿轮上落满了厚厚的灰尘。
一只野猫从管道上跳下来，吓了苏晴一跳。"
```

### 13.8 常见问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| AI生图与剧本描述不符 | 描述不够具体 | 用"五要素法"：谁+在哪+做什么+什么表情+什么光线 |
| 角色换了一张脸 | 角色描述不一致 | 建立"角色视觉锚定词"，每场重复关键特征 |
| 动作生硬不自然 | 描写太笼统 | 拆解动作到"关节级别"：哪个部位、什么方向、多大幅度 |
| 场景缺乏氛围 | 只写了"是什么"没写"感觉怎样" | 加入"氛围词"：阴冷的、温暖的、压抑的、梦幻的 |
| 镜头太单调 | 只用中景 | 每3-4个场景设计一次特殊镜头（特写/俯拍/跟拍） |
| 复杂场景AI无法生成 | 超出AI能力 | 简化到"2人+1环境+1动作"的组合 |

### 14.8 常见问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 12集之间剧情断裂 | 缺乏整体规划 | 先完成大纲再逐集展开，每集开始前回顾上集 |
| 角色越写越走样 | 长文本角色遗忘 | 每集生成前重新发送角色"一句话标签"和"经典台词" |
| 后几集质量下降 | 疲劳+创意枯竭 | 换个AI工具生成，用新视角激发灵感 |
| 伏笔忘记回收 | 没有记录 | 维护"伏笔清单"，埋一个记一个 |
| 爽点分布不均 | 缺乏可视化 | 画出情绪曲线图，直观检查 |
| 对白风格不统一 | 多工具生成导致 | 最后统一用豆包做一次对白风格校准 |
| 总字数失控 | 每集没控字 | 严格要求"每集600-800字"，超出就删 |
