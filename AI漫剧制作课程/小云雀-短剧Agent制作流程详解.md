### 2.2 Step 1：上传剧本与需求确认

#### 2.2.1 剧本格式要求

**支持格式**：标准 `.docx` 文件

**剧本结构要求**：
```
# 剧名：《代码之外》

## 角色表
- 林晓：25岁，黑色短发，程序员，性格内向但倔强
- 苏瑶：22岁，银白色长发，设计师，温柔但有主见
- 赵天：35岁，公司高管，阴险狡诈

## 第1集：困局

### 场景1：出租屋 · 深夜
林晓坐在电脑前，疲惫地敲着代码。
旁白：凌晨两点，林晓还在改一个他写了三天的bug。
林晓（自言自语）：这个bug到底在哪...

### 场景2：公司走廊 · 白天
林晓端着咖啡走向工位，同事小胖迎面走来。
小胖：晓哥，老板找你。
林晓：好，马上来。

### 场景3：老板办公室 · 白天
老板坐在办公桌后，表情冷漠。
老板：林晓，你负责的AI助手项目，公司决定停掉。
林晓：什么？！可是我...

### 场景4：公司天台 · 黄昏
林晓独自站在天台栏杆边。苏瑶走过来。
苏瑶：你还好吗？
林晓：我...不知道该怎么办。
苏瑶：也许，这是个新的开始呢。
```

**格式要点**：
- ✅ 使用标准docx格式
- ✅ 明确标注角色信息（姓名+外貌+性格）
- ✅ 每场标注场景+时间
- ✅ 对白和旁白清晰区分
- ✅ 有完整的场次划分
- ❌ 不支持纯分镜表格
- ❌ 不支持一句话故事输入

#### 2.2.2 上传操作

1. 打开小云雀APP或网页版（xyq.jianying.com）
2. 选择 **"短剧Agent"** 功能入口
3. 点击 **"上传剧本"**
4. 选择准备好的 `.docx` 文件
5. 等待系统解析（通常1-3分钟）

#### 2.2.3 需求确认

上传后系统会要求确认以下设置：

| 设置项 | 选项 | 推荐 |
|--------|------|------|
| **画风** | 2D动漫 / 3D动漫 / 仿真人 | 漫剧推荐2D动漫 |
| **叙事形式** | 沉浸式漫剧 / 解说节奏 | 漫剧推荐沉浸式 |
| **对白处理** | 保留原对白 / 智能改编 | 建议先保留原对白 |
| **目标平台** | 抖音/B站/红果 | 根据发布平台选择 |

### 2.3 Step 2：故事蓝图（核心价值）

这是整个流程含金量最高的环节，Agent自动完成以下工作：

#### 2.3.1 角色自动识别与资产生成

Agent会通读剧本，自动完成：

1. **识别所有角色**：从剧本中提取每个出场角色
2. **分析角色信息**：从剧本描述中提取外貌、服装、性格等
3. **生成形象资产**：为每个角色生成多角度、多表情的参考图
4. **追踪形象变化**：如果角色在剧情中有服装/外貌变化，生成对应阶段的资产

**示例——角色资产自动生成**：

```
角色：林晓

资产1：日常状态
- 服装：深灰色连帽卫衣 + 黑色牛仔裤
- 发型：黑色短碎发
- 表情：平静/疲惫
- 对应场次：第1集 场景1-3

资产2：创业状态（后期）
- 服装：白色衬衫 + 深蓝色西装外套
- 发型：黑色短发（稍长）
- 表情：坚定/自信
- 对应场次：第5集起
```

#### 2.3.2 故事蓝图检查

蓝图生成后，需要检查以下内容：

- [ ] 角色是否全部识别？（有没有遗漏的配角）
- [ ] 角色形象是否准确？（发型、服装是否符合剧本描述）
- [ ] 形象变化是否合理？（角色成长对应的外貌变化）
- [ ] 场次标注是否正确？（每个资产对应的出场场次）
- [ ] 角色关系是否清晰？（谁和谁是什么关系）

**如果发现问题**：
- 角色形象不准确 → 手动修改角色描述，补充参考实体信息
- 遗漏角色 → 手动添加角色信息
- 形象变化不对 → 手动调整不同阶段的资产

#### 2.3.3 参考实体信息补充

如果生成的角色形象不准确，可以通过"补充参考实体信息"来修正：

```
需要补充的信息：
1. 角色更详细的外貌描述（发型长度、发色深浅、面部特征）
2. 具体的服装描述（品牌风格、颜色搭配、配饰细节）
3. 参考图片（如果有手绘或AI生成的概念图，上传作为参考）
4. 角色性格关键词（影响表情和姿态）
```

### 2.4 Step 3：故事板编辑与视频生成

#### 2.4.1 故事板

故事板是每场戏的可视化预览，包含：
- 每个镜头的画面预览
- 角色位置和动作
- 对白/旁白文字
- 镜头类型（特写/中景/远景）

**可以跳过编辑直接生成视频**，也可以手动调整：
- 修改镜头构图
- 调整角色位置
- 修改画面描述
- 调整镜头顺序

#### 2.4.2 视频生成

点击"生成视频"后，系统按集按场次推进：

| 特性 | 说明 |
|------|------|
| 并行处理 | 同时处理3-4场戏 |
| 单次最长 | 8分钟连续内容 |
| 模型 | Seedance 2.0 |
| 输出规格 | 1080P / 30fps |

**生成时间参考**：
| 场次 | 预计时间 |
|------|---------|
| 1-3场 | 5-15分钟 |
| 4-8场 | 15-30分钟 |
| 整集（10-15场） | 30-60分钟 |
| 整部（60集） | 8-12小时（可过夜生成） |

**最佳实践**：
> "白天把分镜全跑完，睡前把生成视频全点一遍，第二天早上起来视频都出来了。" —— 内测用户

#### 2.4.3 连贯性问题处理

如果生成的视频出现连贯性偏差：

```
问题原因：剧本解析阶段的参考实体信息不够完善

解决方案：
1. 对应页面补充完善相关参考实体信息
2. 补充角色外貌的详细描述
3. 补充场景的详细描述
4. 重新生成该场次的视频
```

## 三、提示词与剧本优化技巧

### 3.1 剧本编写提示词（面向小云雀优化）

#### 3.1.1 角色描述模板

```
【角色名】：林晓
【年龄】：25岁
【性别】：男
【发型】：黑色短碎发，略微向右偏分
【眼睛】：深棕色，棱角分明
【体型】：中等身材，178cm
【常服】：深灰色连帽卫衣 + 黑色牛仔裤 + 白色运动鞋
【性格关键词】：内向、倔强、技术宅、不善社交
【标志性特征】：左眉上方有一道浅浅的疤痕
```

**要点**：
- 越详细越好，Agent会根据描述生成资产
- 标志性特征有助于跨场次一致性
- 性格关键词影响表情和姿态

#### 3.1.2 场景描述模板

```
【场景】：林晓的出租屋
【时间】：深夜（凌晨2点）
【空间】：约15平米的单间
【关键元素】：
  - 书桌（靠窗，有笔记本电脑）
  - 单人床（深蓝色床单）
  - 台灯（暖黄色光源）
  - 窗户（可见城市夜景）
【氛围】：昏暗、孤独、疲惫
【光线】：台灯暖黄色 + 电脑屏幕蓝白色
```

#### 3.1.3 动作与情绪标注

```
❌ 不够详细：林晓很累
✅ 足够详细：林晓一只手撑着下巴，眼睛半闭，偶尔揉一下太阳穴，
   屏幕的蓝白色光照在他疲惫的脸上

❌ 不够详细：林晓很震惊
✅ 足够详细：林晓的眼睛猛地睁大，嘴唇微微张开，
   手中的咖啡杯不自觉地倾斜，咖啡险些洒出
```

### 3.2 画风选择指南

| 画风 | 适用题材 | 效果 |
|------|---------|------|
| **2D动漫** | 玄幻、都市、校园、恋爱 | 最常用，效果稳定 |
| **3D动漫** | 科幻、机甲、游戏 | 视觉冲击强 |
| **仿真人** | 都市、悬疑、现实题材 | 接近真人短剧 |

**2D动漫风格细分**：
- 日系动漫：大眼睛、精致五官
- 国漫风格：相对写实、东方审美
- Q版/萌系：可爱风格、适合搞笑题材

### 3.3 叙事形式选择

| 形式 | 特点 | 适用场景 |
|------|------|---------|
| **沉浸式漫剧** | 角色对白+画面，无旁白解说 | 正剧、情感剧、动作剧 |
| **解说节奏** | 有旁白解说推动剧情 | 解说类漫剧、快节奏爽剧 |

**建议**：
- 追求故事沉浸感 → 沉浸式漫剧
- 追求快节奏+信息密度 → 解说节奏
- 可以混合使用（部分场景沉浸式，部分场景解说）

### 4.3 从0到1制作流程（以都市逆袭题材为例）

#### 第一步：准备剧本（2-4小时）

用DeepSeek/文心5.0生成完整剧本，格式参考2.2.1节。

**提示词（给DeepSeek）**：
```
请为我生成一部12集都市逆袭AI漫剧的完整剧本。

【格式要求】
- 使用标准docx兼容格式
- 包含角色表（姓名+外貌+性格）
- 每集包含多个场景
- 每场景标注：场景名+时间+地点
- 对白格式：角色名：「台词」
- 旁白格式：旁白：内容

【剧本内容】
主角林晓，25岁程序员，被裁员后创业逆袭。
12集，每集4-6个场景，总字数约8000字。
```

#### 第二步：上传到小云雀（5分钟）

1. 将剧本保存为 `.docx` 格式
2. 打开小云雀 → 短剧Agent → 上传剧本
3. 选择画风：2D动漫
4. 选择叙事形式：沉浸式漫剧
5. 等待系统解析

#### 第三步：检查故事蓝图（15分钟）

1. 检查角色是否全部识别
2. 检查角色形象是否准确
3. 如有问题，补充参考实体信息
4. 确认无误后进入下一步

#### 第四步：生成视频（1-3小时）

1. 可以直接点击"生成视频"（跳过故事板编辑）
2. 系统按集按场次推进
3. 可以同时处理3-4场
4. 等待生成完成

#### 第五步：后期调整（1-2小时）

1. 检查每场视频质量
2. 不满意的场次重新生成
3. 用剪映进行最终剪辑
4. 添加BGM、音效、字幕
5. 导出1080P成片

**总时间估算**：约5-10小时完成一集（含剧本创作）

## 五、进阶技巧

### 5.1 提升角色一致性的方法

1. **剧本中角色描述要详细**：越详细，Agent生成的资产越准确
2. **补充参考实体信息**：如果形象不准确，手动补充外貌细节
3. **上传参考图**：如果有手绘或AI生成的概念图，上传作为参考
4. **保持描述一致**：同一角色在不同场景中的外貌描述要一致

### 5.2 提升视频质量的方法

1. **剧本质量是根本**：详细的场景描述+动作描写 → 更好的视频
2. **镜头描述要具体**：在剧本中标注镜头类型（特写/中景/远景）
3. **避免复杂多人同框**：2人以内的场景效果最好
4. **动作描写要可执行**：避免"精妙绝伦的剑法"这种抽象描述

### 5.3 批量生产技巧

1. **系列剧管理**：Agent自动维护多集角色资产的一致性
2. **并行生成**：利用3-4场并行处理能力，合理安排生成顺序
3. **过夜生成**：白天准备剧本和蓝图，睡前点生成，第二天收片
4. **模板复用**：成功案例的剧本格式和角色设定可以复用到新项目

### 5.4 常见问题与解决

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 角色形象不准确 | 剧本描述不够详细 | 补充参考实体信息，上传参考图 |
| 视频连贯性差 | 角色/场景信息不完整 | 完善蓝图中的角色和场景描述 |
| 超过2人场景效果差 | AI多角色处理能力限制 | 拆分为多个2人以内场景 |
| 排队时间长 | 用户量激增 | 避开高峰期，或使用Draft模式 |
| 剧本解析失败 | 格式不符合要求 | 确保使用标准docx格式 |
| 画风不统一 | 多次生成时参数变化 | 同一批次使用相同画风设置 |
